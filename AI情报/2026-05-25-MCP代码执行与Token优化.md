## 📡 今日信号

### 1. Anthropic 发布「Code Execution with MCP」工程模式

- **事件**：Anthropic 官方工程博客发文，提出 MCP 工具不应作为直接 tool call 加载到 context，而应暴露为代码 API，让 Agent 写代码来调用。
- **为什么重要**：当 Agent 连接数十个 MCP Server、数千个工具时，tool definitions 会消耗 15 万+ tokens，严重影响效率。新模式下 Agent 通过文件系统按需加载工具定义 + 在执行环境中处理中间结果，token 从 150k 降至 2k。
- **可行动的点**：① 审视你现有 MCP Server 的工具数量，评估 Code Execution 模式的收益；② 在 MCP Server 中增加 `search_tools` 接口实现渐进式发现；③ 利用执行环境做数据过滤/脱敏，避免敏感数据流入模型上下文。
- **来源**：[Anthropic Engineering Blog](https://www.anthropic.com/engineering/code-execution-with-mcp)

## 💡 今日知识点：MCP Code Execution 模式 ⭐

- **知识点名称**：MCP Code Execution（代码执行替代直接工具调用）
- **核心原理**：
  1. 传统模式：所有 MCP tool definitions 塞入 context window → Agent 直接调用 → 中间结果回流 context → 二次消耗
  2. Code Execution 模式：MCP Server 暴露为文件系统上的代码 API → Agent 写代码按需 import → 中间数据在执行环境中处理 → 只返回必要结果
  3. 关键收益：渐进式发现（Progressive Disclosure）+ 数据预过滤 + 隐私保护（敏感数据不经过模型）+ 状态持久化 + 可复用 Skills
  4. Anthropic 基准测试：150k tokens → 2k tokens，降幅 98.7%
- **实际应用场景**：当你的 Agent 连接 3+ 个 MCP Server、工具总数超过 50 个时，应考虑迁移到 Code Execution 模式。特别适合涉及大文档/数据表的场景（过滤 10000 行表格只返回 5 行给模型）。
- **面试话术**：「在我们的 MCP 架构中，我采用了 Code Execution 模式替代直接 tool call——Agent 写代码按需加载工具定义，在执行环境中做数据过滤，token 消耗从 15 万降到 2000。」
