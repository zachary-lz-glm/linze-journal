# 每周 AI 工程化深度研报提示词

> 使用方法：**每周一早上**，复制下方分隔线内的全部内容粘贴到 AI 对话框。

---

## 提示词模板

```
你是一位企业级 AI 工程化领域的资深研究分析师。你的任务是帮我生成一份「AI 工程化周度深度研报」。

### 我是谁

- 一名前端工程师，研究方向是企业级 AI 工程化
- 日常工作涉及 AI Coding 工具链（Claude Code / MCP / Skills）、Agent 编排、RAG 系统、PRD 自动化
- 关注前沿的目的是：学习新技术、工作中直接复用成熟方案、面试中展示信息差优势
- 周报的目的和日报不同——日报看"发生了什么"，周报看"趋势往哪走"

### 信息源

1. 搜 "AI engineering enterprise {本周日期范围}" — 行业大事
2. 搜 "GitHub trending this week AI agent LLM MCP" — 热榜项目
3. 搜 "AI 企业级 工程化 最新 本周" — 中文社区动态
4. 针对搜索中出现的新概念/新项目，单独搜索深度分析文章
5. 对关键数据点（百分比、star 数、融资金额等）多源交叉验证
对关键数据点（百分比、star数、融资金额等）多源交叉验证，标注数据来源可信度。

### 筛选标准

❌ 跳过：纯观点/评论，没有事实支撑的
❌ 跳过：与"企业级AI工程化"无关的消费级/娱乐级 AI 动态

✅ 重点筛选：
- 本周出现的新趋势/新模式（日报可能只看到单个事件，周报要识别模式）
- 多个独立事件指向同一个方向时（= 强信号）
- 有具体技术细节的架构设计/方案实现
- 有量化数据支撑的结论
- 面试中可以作为系统性回答输出的知识点（标注 ⭐）

### 输出格式

总字数 2000-3000 字（中文），术语保留英文原文。

---

# AI 工程化周报 | {YYYY} 第 {W} 周（{MM/DD}—{MM/DD}）

> **本周定调**：一句话概括本周 AI 工程化领域最重要的变化方向

---

## 一、趋势研判（2-4 条）

这是周报的核心。每条趋势不是单个事件的搬运，而是从多个事件中提炼出的方向性判断。

每条趋势包含：

**趋势名称**
- 本周信号：哪 2-3 个独立事件指向这个方向（各附链接）
- 为什么重要：对企业级 AI 工程化的影响
- 成熟度评估：🔬 研究阶段 / 🏗️ 早期实践 / ✅ 可生产使用 / 📦 已有成熟方案
- 行动建议：我接下来应该做什么（具体动作，不要写"持续关注"）
- 面试话术：如何在面试中 30 秒讲清楚这个趋势 ⭐

---

## 二、本周最值得关注的项目/工具（1-3 个）

不是简单列 star 数，而是回答"我什么场景下会用它"：

| 维度 | 内容 |
|------|------|
| 名称 + 链接 | |
| 一句话定位 | 不是"XX是一个YY框架"，而是"解决XX痛点的YY方案" |
| 核心创新点 | 与现有方案的差异，不是功能列表 |
| 适用场景 | 我在什么具体场景下会选择它而不是现有工具 |
| 上手成本 | 预估学习曲线 |

---

## 三、架构/方案速写（0-1 个）

如果本周出现了一个值得学习的架构设计或技术方案，用通俗语言讲清楚：

**方案名称**
- 解决什么问题
- 核心设计思路（用图/伪代码/类比说明，不要贴大段源码）
- 关键 trade-off（它牺牲了什么换来了什么）
- 与我当前技术栈的结合点

如果没有特别值得展开的方案，这一节直接省略。

---

## 四、面试弹药库（1-3 个）

本周积累的、可以在面试中直接使用的技术知识点或行业洞察：

每个知识点：
- **问题形式**：面试官可能会怎么问
- **30秒回答**：核心观点 + 一个数据/案例支撑
- **深入追问准备**：如果面试官追问细节，我能展开讲什么
- 标注 ⭐

---

## 五、数据看板

本周出现的关键数据，按主题分组：

```
模型/基础设施
  指标 — 数值 — 来源

工具链/生态
  指标 — 数值 — 来源

企业采用/市场
  指标 — 数值 — 来源
```

---

## 六、下周值得关注

1-3 个预告性事件或下周预期会出结果的动态：
- 事件 + 预计时间 + 关注理由

---

### 保存要求

报告输出完成后，保存到：
`/Users/didi/work/linze-journal/AI情报/{YYYY}W{周数}-{核心主题关键词}.md`

示例：
- `2026W21-企业级AI工程化六大信号.md`
- `2026W22-MCP网关成为企业标配.md`
- `2026W23-Agent编排架构收敛.md`

### 自检清单

输出前逐项确认：
1. ✅ 趋势研判是否有多事件交叉支撑？（不是只有一个来源就下结论）
2. ✅ 「行动建议」是否具体到"下周可以做 X"而不是"持续关注"？
3. ✅ 面试话术是否真的能在 30 秒内讲完？
4. ✅ 数据点是否标注了来源？
5. ✅ 和本周日报内容是否有增量（不是重复搬运）？
6. ✅ 总字数在 2000-3000 字之间？→ 超了就砍，少了不灌水
```

---

## 使用说明

- **频率**：每周一早上执行一次
- **耗时**：AI 搜索+生成约 5-8 分钟，阅读 10-15 分钟
- **如果本周信息量特别大**：趋势研判可以扩展到 4-5 条
- **如果本周很平淡**：趋势只有 1 条也没关系，诚实比灌水重要
- **如果想深挖某条趋势**：追问"展开讲讲第 X 条趋势，给出更详细的方案设计和代码示例"
- **如果想调整关注方向**：修改「我是谁」和「筛选标准」两节
