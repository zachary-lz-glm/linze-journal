**30 秒面试开场：**

> 我在滴滴国际化做营销中台，核心是 Schema 驱动架构——BFF 层用模板引擎驱动 Schema 生成，字段定义按模板分布式管理，通过 injectDActions 声明式注入联动配置（一次声明双重求值：BFF 端 safeEval 设初始状态 + 序列化 d_actions 给前端运行时联动）。联动分 fetchApi 型回调 BFF 重生成和 state 型前端本地 eval。前端 Schema 渲染引擎拿到后通过 formatName 将嵌套表单拍平为 dot 路径，通过 Listener 发布订阅执行联动，35+ 组件动态渲染。这套 Schema 驱动模式不只是活动表单在用，权益组件库（benefit SDK）也是 Schema 消费者——24 个内置组件通过 Register 注册，后端返回的 form_data 同样用 d_component 映射 + FieldWrapper 迭代渲染，两个消费者共享同一套架构。在此基础上做了 prd2code-gen，用 AI 半自动化 BFF 开发。
