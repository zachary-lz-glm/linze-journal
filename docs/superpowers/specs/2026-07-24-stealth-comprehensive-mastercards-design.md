## 1. 背景与目标

### 1.1 为什么现在综合卡片"不好用"

诊断(已与用户对齐):

1. **分组按"面试环节"而非"大问题"**。现 emer 按 `项目数据/HR面/软技能/反问/话术急救` 分,但用户真正要的 9 个大问题(看机会/竞争力/职业规划/offer/创业/成就感/业务指标/最大困难/行业洞察)**横跨这些组**,且其中 4 个核心答案(离职原因/职业规划/精力分配/动机收尾)根本不在综合 tab、在 `open` 开场 tab。
2. **答案只有一层,扛不住深挖**。多数卡是单层表面答案,被 5-Why / "105 道追问" / progressive questioning 式追问就散。这正是用户引的四篇大佬方法论(掘金证据链 / alexeygrigorev progressive questioning / CLEAR+Reflection / MyEngineeringPath 11段)要治的病。
3. **缺少"预判追问"层**。现有卡答完表面问题就停,没把"面试官最可能追什么、我怎么接"显式写出来。

### 1.2 目标效果(复刻项目主卡重构)

照搬 2026-07-21 项目主卡重构的套路(给每个项目 1 张五步主陈述卡 + 机制弹药卡),搬到**个人叙事**:

- 综合 tab 脊梁改成 **9 张「自我陈述主卡」**,每张 = `core`(30秒钩子) + **CLEAR/L 答案**(Context/**Leadership**/Execution/Results) + **证据锚点** + **诚实边界(Reflection)** + **内嵌预判追问清单**。
- 把现有散落的相关卡**并入对应主卡**(内容吸收后删独立卡),卡数从 28 降到 22,更利索。
- 真正"非叙事"的实用卡(话术急救/反问/项目数据/部分HR)留作**弹药·参考**,主卡的追问清单"→ 指向"它们。

### 1.3 方法论来源(用户指定)

- **CLEAR**(STAR 升级版) = Context/**Leadership(为什么是你)**/Execution/Accomplishment-Results。L 是它相对 STAR 的精华。⚠️ 带 Reflection 的是 CARL/STAR+L,本 spec 的"诚实边界"段即 Reflection 落点。
- **掘金·证据链**:AI 时代面试像证据审查,材料要"可追问、可验证、可复盘"。→ 主卡的"证据锚点"段 + 预判追问。
- **alexeygrigorev·progressive questioning / "eval or vibes" / "acknowledge gaps still get hired"**:→ 追问链 + 诚实边界。
- **zsc·5-Why / MyEngineeringPath·105 追问**:→ 每张主卡预判 4-7 条追问。
- 讲故事方法论 memory:五步骨架、tentpole、三段推理链(选型:考虑A/B/C→选B因为→代价是→用D补偿)、诚实边界4步。

## 3. 架构决策

### 3.1 双层结构(同项目重构)

- **自我陈述主卡**(本次新建,9 张):开场/HR/行为面被让展开时用,套 CLEAR/L + 预判追问。
- **弹药·参考**(现有实用卡,保留):被追问单点(话术/反问/数据)时翻;主卡追问清单"→ 指向"它们。

### 3.2 9 张主卡 = 9 大问题(用户清单),每问 1 张

| # | 大问题 | 主卡标题(序号①-⑨) | 并入的现有素材 |
|---|--------|------|------|
| ① | 看机会原因 | 为什么看机会/为什么离开滴滴 | `open`离职原因 + `emer`空窗期 + `open`动机收尾 |
| ② | 核心竞争力·优缺点 | 你的核心竞争力/优缺点 | `emer`优点和缺点 + `emer`为什么录用你 |
| ③ | AI时代·职业规划 | AI时代的职业规划/你想做什么 | `open`职业规划 + `open`精力分配 + `emer`AI会取代前端吗 |
| ④ | offer·谈判 | 手里offer/薪资怎么谈 | `emer`手上有其他offer吗 + `emer`薪资低于预期 |
| ⑤ | 对创业的想法 | 对创业的想法/想要什么状态 | `emer`创业3卡(为什么想要创业状态/从0到1兴奋/为什么主动选创业节奏) |
| ⑥ | 最有成就感 | 最有成就感的事 | `emer`最有成就感的事 |
| ⑦ | 业务指标关注度 | 你关注业务指标吗/怎么挂钩 | `emer`价值靠什么衡量 + `emer`业务方关心什么 |
| ⑧ | 最大困难·怎么解决 | 最大的困难/怎么解决的 | `emer`最大的失败挫折 + `emer`方案推不动 |
| ⑨ | 行业洞察·深度了解 | 行业洞察/你深度了解过哪些方案 | 引用`emer`反问4卡洞察(快手/动念/B端/同类工具,**卡留作弹药**) + 并入`emer`了解X吗 |

> ③ 的"想做什么"由职业规划+创业卡推导(见 §6.3),用户 review 时拍板。④ offer 默认通用话术,用户给真实状态再改。

### 3.3 内嵌预判追问清单(已与用户对齐)

每张主卡 body 末尾嵌「预判追问」区:列 4-7 个面试官最可能追的问题,每条配**一句话答**或"→ 指向某弹药卡"。主卡仍 1 张、手机可扫;追问有覆盖但不臃肿。这是项目主卡⑤方法论 + 现有「被问·我答」hint 的自然延伸。

### 3.4 风格:口语逐字稿(非 §6 declarative)

- 综合是 HR/行为面,说人话。对标现有**创业3卡**(model):第一人称、自然连接词、能直接念。
- **不用**项目卡那套 §6 declarative(`<strong>段首。</strong>` + ①②堆叠)——那是技术机制卡的规矩。
- 但避 §1.1 碎嘴滥用:少滥用 `<strong>`,不用"必背/杀手锏/被问X必答"速查表腔。

### 3.5 open tab 不动,主卡自包含

`open` 的离职/职业规划/精力/动机**保留原位**(开场流不能断)。综合主卡做**自包含深版**:body 复述核心答案(套 CLEAR/L)再叠加追问链/证据/边界;不依赖 open 才能读懂。可接受与 open 轻微内容重叠(两 tab 各自连贯、可搜)。

## 4. 事实校准策略(防穿帮,§1.3 铁律)

### 4.1 Q9 深度了解分档(用户已勾选,关键防穿帮)

每提一个框架**主动报水位**,别等被追降档(opendesign一面血泪):

- **项目真用过/自研实现**(撑得住"讲讲实现"):**LangGraph**(Manager/Code 项目 ReAct 双节点/Supervisor 编排)、**向量检索·RAG**(text-embedding-v3 + 自研内存余弦 + sha256 增量)、**GraphRAG 思路**(prd-tools 轻量实体图,受启发**非图数据库非 AST**)。
- **读过源码**:**OpenHands**(原 OpenDevin,MIT/文档全,读过 agent runtime 和事件流)——撑"看过哪个源码"这档。
- **产品体验级(老实报档,非源码)**:**Cursor / Claude Code**(闭源,能讲交互设计判断)、**OpenAI Agent SDK**(360 面试官提过他们用)、**MCP 协议**(近热门,了解协议定位)。卡里明确标"产品体验",绝不装"读过源码"。

### 4.2 Q8 主线故事(用户已选)

**推广 AI 工作流遇阻力**(改变一群人工作方式)为主线深挖,与⑥成就感/⑤创业一条线。yarn 事故 / Schema 中台 / 推广不理想 作追问备用。

### 4.3 数字沿用已校准值

- 创业卡项目数字沿用已校准:Manager 39 节点状态机(qwen-flash)、code_assistant 18 工具/只读收敛到 11(qwen-coder-plus)、prd-tools 迭代到 v2.20。
- prd-tools:5h→2h(n=5 小样本)、40%→85%(n=10 产物稳定性非质量)、团队模式准确率损耗约 60%——均带口径,可疑即软化。
- **雷区不碰**:Manager 非 Claude/GPT、12 Agent=9外+3内、失败归因纯规则、自动晋级默认关;Code 向量自研非 Chroma、增量靠 sha256 非 git 钩子、无 Diff 预览确认/独立审查 Agent/HITL 写确认;PRD prd-tools 真身是 Claude Code 插件、"6 步闭环"不存在、"20+人"软化(ADR-0005 写 1-5 人)。
- 业务指标数字(完单率~70%、留存30日~30%、LTV/CAC>3)是行业常识级,念时带"行业大概"。

## 5. 主卡模板:CLEAR/L + 证据 + 边界 + 追问清单

HTML 沿用 GUIDE §2.2 卡片模板,`data-c="emer" data-g="自我陈述"`。结构:

```html
<div class="card" data-c="emer" data-g="自我陈述" data-kw="关键词 覆盖各种问法">
  <div class="card-head"><span class="tag emer">短标签</span><div class="card-title">①核心标题(不带项目名)</div></div>
  <div class="core">一句口语化核心(金色,30秒钩子)</div>
  <div class="card-body">
    <p>【背景+为什么是你】Context + Leadership……</p>
    <p>【我怎么做的】Execution……</p>
    <p>【结果+证据锚点】Results + 具体证据(项目/数字/观察,可追溯)……</p>
    <p>【反思/诚实边界】Reflection:局限、没做成、下次怎么做不同。acknowledge gaps still get hired。</p>
    <p>【预判追问】</p>
    <ul class="points">
      <li>追问1?→ 一句话答 / → 指向弹药·XX卡</li>
      <li>追问2?→ ……</li>
      <!-- 4-7 条 -->
    </ul>
  </div>
</div>
```

- `core` 先念(30 秒钩子,对应 MyEngineeringPath「30-Second Version」);body 是被让展开的 1-2 分钟版。
- 4 段 CLEAR/L 齐全,**Leadership 段**(为什么是你/你的主动选择)和**Reflection 段**(诚实边界)是本次重点,非空话。
- 证据锚点段必须挂可追溯证据(项目机制/已校准数字/真实观察),不是 vibes。
- 预判追问用 `<ul class="points">`,每条"问?→ 答/指向"。

## 6. 九张主卡内容骨架

### 6.1 ① 为什么看机会/为什么离开滴滴
- **core**:4 年从 0 到 1 验证完了,想带这些能力去更专注 AI、AI 是业务生命线的场景(⚠️ 不说"更大平台",字节当面怼过站不住)
- **C+L**:滴滴 4 年不差,中台从 0 到 1 我主导、连续 A。离开是主动选不是逃。L=框框我定够了,想要下一个 0 到 1,这步是我主动要的。
- **E**:两件事触发:①中台跑稳,0→1 阶段走完、曲线变平;②想全职做 AI 但岗位主线还是前端,Spec-Driven 是挤时间推的,往下做深(评测/Multi-Agent 生产化)需要 AI 主线 + 真实流量场景。离职后做了两件事:梳理开源(Schema 驱动 + AI 工程化沉淀成工具)+ 体验市面 AI Coding 产品建行业感知。
- **R+证据**:连续 A 绩效、中台多团队在用、个人 Agent 项目(39 节点/18 工具)。
- **边界**:空窗是主动选不是被动空着;不吐槽、不提钱不提加班;别说"更大平台",说"AI 是业务生命线的场景"更具体。
- **追问**:为何不留滴滴做 AI?→ 岗位主线是前端,AI 是挤时间 / 为何现在看?→ 工具开源方法验证完了,该带去大场景 / 空窗在干嘛?→ 开源 + 体验 AI 产品建感知 / 你来了也走怎么办?→ 我要的是 AI 主线,你们正好是,匹配不是过渡。

### 6.2 ② 你的核心竞争力/优缺点
- **core**:稀缺组合 = 前端架构 + AI 工程化双背景;大多数前端不懂 AI 工程化、大多数搞 AI 不懂前端工程
- **C+L**:优点不是形容词,是组合稀缺性。L=我有什么别人没有的组合。
- **E**:优点①从业务抽共性做架构(Schema 覆盖十几种活动、新增零代码)②全流程质量把控(prd-tools 证据链 + 门禁,不是让 AI 写完就完)。缺点说真的:架构有时过度追求通用性——联动引擎初期搞灵活嵌套联动,80% 场景就是简单显隐/值联动,复杂用例反增理解成本;后简化成标准模板覆盖绝大部分、超范围才自定义。
- **R+证据**:Schema 中台(多团队用、上线 2 周→3 天)+ Agent 项目(39 节点/18 工具)+ prd-tools(5h→2h n=5、40%→85% n=10)。缺点带补法:架构服务实际场景、不为架构而架构。
- **边界**:专业短板说"AI 工程化的 eval 体系还不成熟,正在补"——边缘性缺点 + 补法,别碰岗位核心。
- **追问**:独特在哪?→ 组合稀缺不是单一技能 / 缺点细说?→ 联动引擎过设计例子 / 最不擅长?→ eval 体系在补 / 前端框架你最强?→ 诚实:强在架构抽象不在框架细节。

### 6.3 ③ AI 时代的职业规划/你想做什么
- **core**:做深 Agent 工程化 → AI 工程化技术负责人;中期想把 AI 工程化做成能往团队推的平台级能力、own 一条从想法到落地的闭环;不追风口追工程深度
- **C+L**:1 年补评测(Manager 自进化每臂样本太少,做不到统计显著 → 要在有真实流量场景补 Agent 评测);3 年 Multi-Agent 生产化(从业余 39 节点 demo 推到生产级:稳定性/成本/可演进);5 年技术负责人(带 5-10 人、对 Agent 平台技术路线负责)。L=前端 owner 让我懂对业务线负责、Spec-Driven 让我懂对平台负责,下一台阶对技术方向负责。
- **E**:**想做的事** = 把 AI 工程化从"个人能用 Cursor"做成"团队/平台级能推的能力"(SSOT + 证据链 + 门控这套)。精力 70% 工作真落地 + 30% 个人 Agent 预研(技术栈延续、反哺工作)。
- **R+证据**:Spec-Driven 团队稳定在用、个人项目证明能做真 Agent(LLM 路由/自主选工具);连续 A 本职没拉下。
- **边界**:个人项目偏预研没接真实流量、Multi-Agent 没真实流量验证;陈哲类创业 CTO 面可主动说想要 owner 身份(→ ⑤创业)。
- **追问**:为什么不转管理?→ 要技术深度不是躲技术 / 评测怎么补?→ 建 labeled query set 测 recall@K/nDCG / Multi-Agent 生产化最大挑战?→ 稳定性 + 成本 / 管理还是 IC?→ 技术负责人(带小队 + 对方向负责) / AI 会取代前端吗?→ 分层:编码能取代,系统设计/质量把控/业务判断取代不了。
- ⚠️ "想做的事"为推导(平台级 AI 工程化能力 + own 完整闭环 + Agent 基建/技术负责人),**用户 review 时拍板**。

### 6.4 ④ 手里 offer/薪资怎么谈
- **core**:适度透露证竞争力,不暴露公司名和数;薪资看综合 package 不只 base
- **C**:手上 offer——有就说"流程中几个机会,两个到终面/offer 阶段,方向都跟 AI 工程化相关,但最看重团队能做的事和匹配度";没有就说"刚开始看,之前在系统整理项目经验和行业认知,贵司是认真准备的第一批"。
- **E**:薪资——先给范围不给死数让对方先出价;低于预期不说"太低",说"比预期低些能聊空间吗";base 谈不动争签字费/期权/年终系数/到岗灵活。
- **R+证据**:默认通用话术(用户真实 offer 状态待填)。
- **边界**:绝不说具体公司名(HR 打听)、具体薪资(暴露底牌)、"没有其他 offer"(丢筹码);底线:低于最低线直接拒,接受不满意 offer 入职后心态出问题。
- **追问**:现在有几个 offer?→ 按真实状态 / 期望多少?→ 给范围不给死数 / 给不到期望?→ 看综合 package / 多久到岗?→ 按实际留灵活 / 为什么选我们?→ 方向匹配 + 做的事正是我想验证的(→ ①)。

### 6.5 ⑤ 对创业的想法/想要什么状态
- **core**:不是能扛是想要——想要 owner 身份:一个东西从想法到落地归我管;大厂给我最后一步是把这层边界拿掉
- **C+L**:创业渴望不是浪漫想象,是已经在过这种日子(prd-tools/Manager/code_assistant 一个人从想法到落地)。L=最想要决策权在我手里,大厂框框是上层定的,我下个台阶想框框也归我定。
- **E(创业3卡收敛为3面)**:①为什么想要创业状态(已在过日子 + 要决策权 + 不确定性是吸引力 + 代价算过主动愿意);②从 0 到 1 兴奋在哪(Schema 中台从自己痛点长成别人在用的基础设施 + 周末 Agent 项目同一爽法 + 维护稳定系统提不起劲 = 离开大厂稳定最诚实原因);③为什么主动选创业节奏(反转"你能扛吗"→ 我想要 + 周末项目证据 + sha256/原子写/shadow→A/B 是创业节奏工程化表达"先能错再能快")。
- **R+证据**:周末项目(prd-tools 迭代到 v2.20、Manager 39 节点、code_assistant 18 工具带鉴权沙箱并发锁 A/B 进化)证明学习速度 + 工程化能力(非 demo 是生产级关注)。
- **边界**:个人项目偏预研、没接真实流量没生产用户规模,不假装是生产系统;但证明学习速度 + 工程化能力。
