## 1. 背景与目标

整体重构主轴（roadmap §1）：讲自己项目时填充词、流水账、自我矮化，**讲"做了什么"不讲"为什么决策+代价+反思"**。三件套主卡（9cec423）+ A·P0 老前端 5 张（kb-v324）+ C·P2 ai 14 张（kb-v326）已把**主陈述卡**的五步范式验证有效。

本轮处理 **P1 机制弹药卡**——被面试官追问单点机制/知识点时翻出来用的问答型卡。勘察结论（Explore agent）：这些卡普遍是"机制说明书"，缺 ownership（我为什么这么定）、alternative（拒绝了什么）、eval（怎么验证有效）。被追问"你为什么这么做/换个做法行不行/怎么知道有效"时，卡上找不到答案。

**用户已定"轻档"**：只补 ownership/alternatives/eval 维度，**不动结构、不重组、不重排、不删卡、不加 data-g、不动 subOrder**。弹药卡套整段五步会别扭（A spec 已证），所以本轮是**加段不重写**的轻补。

**目标**：69 张弹药卡按现状分两档，各补缺的维度，让每张在被追问时能多一句"我的取舍 / 怎么验证"，经得起"为什么这么做/代价/换你怎么办"追问。纯聚合卡/速查表不动。

## 5. 补维度模板（两档）

HTML 沿用 GUIDE §2.2 卡片模板 + 现有 `<p><strong>段首领起。</strong>` 约定（不加新 class）。**不改 core、不重写原文好句、只在 card-body 加一小段**。

### 🅰 极薄/纯机制档 — 加"我的取舍"段
在现有 body 末尾追加一小段（1-2 句）：
```
<p><strong>我的取舍。</strong>我在 code_assistant/Manager/PRD 工作流里定的是 X；
本可以 Y（更省事/更通用），但没选，因为 Z。</p>
```
- ownership 挂回"我的决策"，alternative 点出拒绝了什么。
- 不加本质矛盾/方法论弧（那是主卡的事）。

### 🅱 半截 STAR/亮点成熟档 — 加"怎么验证 + 方法论收口"段
在现有 body 末尾追加一小段（1-2 句）：
```
<p><strong>怎么验证 / 沉淀。</strong>效果看 W（或诚实：目前框架先行没度量，若要度量会按 V 做）；
这件事的本质是 U，下次遇到同类问题我会 T。</p>
```
- eval 非 vibes（有指标就说，没有就诚实承认 + 点度量路径）。
- 方法论句拔成可迁移命名判断。

### 硬约束（carryover）
- **只增不删**：改后 body 保留所有原文好句，长度 ≥ 原文。
- **§6 declarative 风格**，禁口语碎嘴（说白了/得说句实话/我自己比较得意/被问X我答/这套东西）。
- **strong 标句首自然词**，不孤立、念着不顿。
- **数字按 §4.1 软化**，不编。
- 不堆①②③④⑤速查表腔。
