## 项目经历

### AI 工程工作流 — PRD 蒸馏与领域知识管理

*独立设计 + 开发*

B 端业务中 PRD 到代码的核心难点是业务词与代码字段不一致、前后端认知分歧、历史经验没有沉淀。独立设计工程化方案，将 PRD 转化为有证据、可执行、可追溯的结构化开发计划。

- 设计双 Skill 体系：知识库构建 Skill（6 种模式：全量构建 / 增量更新 / 健康检查 / 质量门控 / 反馈回流）+ PRD 蒸馏 Skill（8 步工作流），通过反馈回流闭环实现知识库越用越准
- 设计 6 文件 SSOT 领域知识架构，按关注点分离，每个事实只存在一处，跨文件 ID 引用，5 条边界规则避免知识污染
- 设计能力面适配器，前端 / BFF / 后端通过 20+ 能力面统一描述项目能力，不绑定固定目录结构，跨项目可复用
- 建立证据链 + 质量门控体系：8 种证据类型，PRD 入口三级门禁（pass / warn / block），reference 致命项 + 警告项 + 边界检查
- 实现三层图谱融合（代码结构 + 业务语义 + 治理维度），图谱为发现层，reference 为精选知识库，图谱不可用时完全回退源码扫描

### Schema 驱动营销中台 — BFF 动态生成 + 前端动态渲染

*前端架构 + BFF 架构设计*

国际化营销中台需支持 23 种活动类型，每种有不同字段组合和联动规则。设计 Schema 驱动架构，BFF 动态生成 Schema → 前端动态渲染表单，新增活动类型前端零代码。

- 设计 BFF 模板引擎驱动 Schema 生成，字段按模板分布式管理，联动模板从渲染模板提取 handler 复用，保证联动与初始渲染结构一致
- 设计双重求值联动机制：一次声明联动配置，BFF 端求初始状态（SSR 正确），同时序列化给前端运行时联动，支持 fetchApi 型（回调 BFF 重生成）和 state 型（前端本地 eval）两种联动模式
- 设计前端 Schema 渲染引擎：嵌套路径拍平为 dot 路径，注册表支持 35+ 组件动态渲染，发布订阅模式执行联动
- Lerna + Nx + Yarn workspace 管理 15+ 业务模块 Monorepo，独立版本 + 构建缓存 + 依赖治理

### 权益组件库 — 400+ 权益 React 组件库

*架构设计 + 核心开发*

营销平台需管理 400+ 权益全生命周期，权益类型持续增长。设计可扩展的组件架构，支持业务快速接入新权益。

- Register 注册模式：内置 24 组件，支持运行时注册自定义权益组件，无需修改 SDK 源码
- Schema 同构渲染：组件库与活动表单共享 Schema 渲染架构，后端返回 Schema 前端迭代渲染，两个消费者复用同一套架构
- Rollup 双模打包（ESM + CJS + 类型声明），peerDependencies 避免依赖爆炸，Changesets 语义化发版
