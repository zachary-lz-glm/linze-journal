# 面试作战面板

> **这是什么**：每日训练入口，告诉你今天练什么、不要练什么
> **什么时候看**：每天训练开始前看一眼
> **怎么用**：根据今天的目标，打开对应文件开始练习

## 当前定位

**高级前端 / AI 工程型候选人**

主线：

```text
Schema 生成 -> Schema 渲染 -> benefit SDK 同构复用 -> prd-tools AI 工程化
```

不要把自己讲成“纯页面开发”。你的职级杠杆是：

- 做过复杂中台的 Schema 架构
- 懂 BFF 编排和前后端联动边界
- 做过组件库工程化和 Monorepo 治理
- 能把 AI 从 prompt 玩具落到证据链、质量门控和反馈回流

## 投递阶段每日节奏

### 无面试日（投递 + 练嘴 + 算法）

1. 口述答案卡 30 分钟（挑 3-4 张，录音回听）。
2. 刷 1-2 道算法，每题必须讲复杂度。
3. Boss 直聘 / 内推投递 2-3 家。
4. 回复 HR 消息，推进流程。

### 有面试日

1. 面前 30min：只看答案卡核心句 + 急救包。
2. 面试中：副屏开速查手册（Ctrl+F 备用）。
3. 面试后 10min：填 `面后复盘.md`。
4. 面试后当天：补答案卡 + 练卡住的算法。

### 脑内速查（面试带这个）

```text
主线：Schema生成 -> Schema渲染 -> benefit同构 -> prd-tools
字节：why / trade-off / AI / SSE / 算法
快手：工程化 / 稳定性 / Monorepo / 发版
百度：React原理 / 浏览器 / 网络 / 系统设计
美团：HTTPS / 首屏优化 / AI辅助 / RAG
兜底：实际遇到过 -> 核心矛盾 -> 具体做法 -> 如果重做
```

## 今天禁止做的事

- 不再无限扩资料。
- 不背长篇书面答案。
- 不编造没做过的数据。
- 不把 AI 当面试现场隐蔽提词器。
- 不在算法题直接跳最优解，先讲暴力和优化过程。
