# /面试准备

收到面试邀请后使用。自动搜索该公司真实面经，分析 JD 与候选人经验匹配度，生成完整的定向准备文档。

## 用法

```
/面试准备 字节跳动 TRAE团队 一面
```

然后粘贴 JD。也可以一次全给：

```
/面试准备 字节跳动 TRAE团队 一面
JD：[粘贴JD全文]
```

如果还没收到面试，只是想提前准备某公司：

```
/面试准备 DeepSeek Harness团队
```

会搜索该公司的招聘信息并生成准备文档。

## 执行流程

### 第一步：读取候选人背景

读取以下文件获取最新背景：
- `learning/interview/简历.md` — 当前简历
- `learning/interview/规划.md` — 面试规划与目标

提取：当前定位、项目经历、技术栈、核心数据指标。

### 第二步：搜索面经 + JD 信息（必须联网搜索）

同时发起以下搜索（至少 4 组）：

1. `"[公司名] 前端 面经 2025 2026"` — 近期真实面试题
2. `"[公司名] 面试题 [岗位方向]"` — 该方向的高频题
3. 如果是二面或更高轮次：`"[公司名] [轮次] 项目深挖 面经"`
4. `"[公司名] 面试风格 流程"` — 面试流程和风格特点
5. 如果是 AI/Agent 方向：`"[公司名] AI Coding Agent 面试 2026"`
6. `"[公司名] [岗位] 薪资"` — 薪资范围参考

从搜索结果提取：
- 面试流程（几轮、每轮时长、每轮重点）
- 面试风格（深挖原理？场景落地？压力测试？手撕代码量？）
- 高频题目清单
- 该公司偏好的候选人画像
- 薪资区间

### 第三步：JD 深度分析

分析 JD（用户提供或搜索到的），输出 **JD→经验映射表**：

| JD 要求 | 候选人对应经验 | 匹配度 | 话术要点 |
|---------|-------------|--------|---------|
| ... | ... | ★★★★★ | ... |

**匹配度评分标准**：
- ★★★★★：有 1:1 对应的项目经验，可直接讲
- ★★★★：有相关经验，需要桥接话术
- ★★★：间接相关，需要拆解底层能力对应
- ★★：有了解但没深度实战
- ★：完全缺失，需要准备兜底话术

**核心结论**：总结 3 个最强匹配点 + 1-2 个需要准备的薄弱点。

### 第四步：生成定向准备文档

基于面经 + JD 分析，生成完整的定向准备文档，保存到 `learning/interview/定向/[公司名].md`。

文档必须包含以下章节：

#### 一、JD 分析与匹配度
- 1.1 JD 原文要点（岗位职责 + 任职要求）
- 1.2 JD→经验映射表（上面第三步的输出）
- 1.3 面试风格特点（表格：维度/特点/应对策略）
- 1.4 面试流程（表格：轮次/面试官/时长/核心内容/特点）

#### 二、基础高频题
按该公司面经中实际出现的频率排序，分为：
- JavaScript 核心（标注频率：极高/高/中）
- CSS 高频
- React / Vue
- 浏览器与网络

每道题列出：回答要点 + 频率。
如果该公司有特殊偏好的题型（如拼多多爱考 JS 手写），重点标注。

#### 三、手撕代码
如果该公司有手撕代码传统（如拼多多 4+ 道），列出：
- 第一梯队：必考（附完整代码）
- 第二梯队：大厂通吃（附核心思路）
- 第三梯队：偶尔出现（附核心思路）
- 练习优先级

如果该公司不太考手撕代码，简略带过。

#### 四、算法题
基于面经提取该公司常考算法方向和具体题目，标注难度和核心思路。

#### 五、项目深挖 + 场景设计
- 5.1 该公司追问模式（N 层递进表格）
- 5.2 项目话术锚点（对准 JD 关键词）
- 5.3 场景设计题（基于 JD 业务场景出题）
- 5.4 如果是 AI Coding 方向，额外包含 AI 工程深挖题

#### 六、方向专项题
根据 JD 方向自动选择：
- **AI Coding / SDD 方向**：SDD 全流程、Vibe Coding vs 标准化研发、Harness 约束、Agent 框架、RAG、MCP 协议、Prompt Engineering、AI 代码质量管控
- **传统前端方向**：性能优化、工程化、组件库设计、SSR、微前端
- **全栈方向**：BFF 设计、Node.js 中间件、数据库基础、CI/CD

#### 七、软素质 + HR 面
- 7.1 "为什么跳槽"（公式：感恩过去 + 明确诉求 + 匹配该公司）
- 7.2 职业规划（分时间段的话术）
- 7.3 该公司特有问题（基于面经和公司文化）
- 7.4 反问环节（准备 2-3 个，必须和 JD 业务相关）
- 7.5 HR 面注意事项

#### 八、备战优先级

| 优先级 | 内容 | 时间 |
|--------|------|------|
| P0 | ... | 今天 |
| P1 | ... | 明天 |
| P2 | ... | 面试前 |

#### 九、30 秒自我介绍（该公司定制版）
基于 JD 核心关键词 + 候选人最强匹配点，写一段 30 秒自我介绍。

#### 十、面试当天一页纸
用代码块输出一页纸速查，包含：
- 主线锚点
- 手撕必会（如果该公司考）
- 八股核心（5-8 个关键词）
- 项目追问 N 层防御
- 兜底句（3 句）

#### 十一、面试记录模板

| 轮次 | 日期 | 面试官 | 时长 | 结果 | 备注 |
|------|------|--------|------|------|------|
| HR 初筛 | - | - | - | ⏳ 待定 | |
| 一面 | - | - | - | ⏳ 待定 | |
| ... | | | | | |

#### 十二、来源索引
列出所有搜索来源的 URL 和内容概要。

### 第五步：更新面试规划

读取 `learning/interview/规划.md`，在"四、投递追踪"表格中添加该公司记录：

|------|------|------|------|------|------|

## 候选人背景

以实际读取的 `learning/interview/简历.md` 为准，重点关注：
- 当前定位（标题 + 个人优势）
- 核心项目（SDD 体系搭建、Schema 驱动中台、权益组件库）
- 技能栏（AI Coding / SDD & AI 工程 / 前端 & 架构）
- 量化数据（60%+、70%+、400+、20+ 等）

## 硬性原则

- **必须联网搜索面经**，禁止使用旧数据或编造面经
- 搜索至少 4 组关键词，覆盖不同维度
- 匹配度分析必须诚实，缺失的能力标注为 ★ 并提供兜底话术
- 自我介绍和话术必须锚定 JD 关键词，不能泛泛而谈
- 手撕代码和算法题必须附完整代码或核心思路，不能只有标题
- 场景设计题必须基于 JD 业务场景，不能出通用题
- 面试风格分析必须基于真实面经，不能靠猜测
- 如果该公司有已知的文化特点（如拼多多高强度、字节 OKR），在 HR 面部分标注
- 文档保存后告诉用户文件路径，并提示可以配合 `/模拟面试` 使用
