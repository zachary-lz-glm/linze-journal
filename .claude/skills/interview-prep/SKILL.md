# /面试准备

收到面试邀请后使用。**先挖候选人自己的真实面经库**，再联网搜索补缺，分析 JD 与候选人经验匹配度，生成完整的定向准备文档。

## 用法

```
/面试准备 字节跳动 TRAE团队 一面
```

然后粘贴 JD。也可以一次全给：

```
/面试准备 字节跳动 TRAE团队 一面
JD：[粘贴JD全文]
```

如果还没收到面试，只是想提前准备某公司：

```
/面试准备 DeepSeek Harness团队
```

会搜索该公司招聘信息并生成准备文档。

## 知识库结构（执行前必读）

| 资产 | 路径 | 用途 |
|------|------|------|
| 当前简历 | `learning/interview/resume/v3.md` | 候选人最新背景、项目、量化数据 |
| 已有定向文档 | `learning/interview/定向/*.md` | 同类公司的准备成果，可复用 |
| **真实面经·Q&A** | `reviews/qa/*.md` | 候选人亲历的面试问答整理（第一手真题） |
| **真实面经·转写** | `reviews/transcripts/*` | 面试录音转写原始稿（细节、卡壳点） |
| 项目复盘弹药 | `archive/retrospectives/*.md` | 项目深挖时的技术细节来源 |
| 输出位置 | `learning/interview/定向/[公司名].md` | 本次生成的定向文档 |

> 注意：原 `规划.md`、`题库/` 已废弃，不要再引用。投递追踪改为在定向文档内维护（见第五步）。

## 执行流程

### 第一步：读取候选人背景

1. 读 `learning/interview/resume/v3.md`，提取：当前定位、核心项目、技术栈、量化数据指标。
2. 列出 `archive/retrospectives/` 下的复盘清单，标记哪些可作为本次项目深挖的弹药（按 JD 方向筛）。

### 第二步：挖本地真实面经库（核心步骤，先于联网搜索）

这是本 skill 最可靠的信息源——候选人亲历的真题永远比网上面经准确。

1. **精确命中**：在 `reviews/qa/` 和 `reviews/transcripts/` 中查找该公司同名/近名文件（如「拼多多」「阿里一面」）。命中则完整读取，提取：
   - 该公司**实际问过的题**（标注为「✅ 亲历真题」）
   - 候选人当时**答得好的话术**（可复用）与**卡壳/答崩的点**（本次重点补强）
   - 面试流程、面试官风格、轮次节奏的真实细节

2. **跨公司复用**：即使该公司无记录，也扫一遍 `reviews/qa/` 里**同方向或同梯队**的公司（如同为 AI 方向、同为大厂），提取：
   - 候选人**反复被问**的高频题（跨多家出现 = 必准备）
   - 候选人**反复卡壳**的薄弱点（系统性短板，必须解决）
   - 已验证有效的项目话术锚点

3. 产出一份「本地命中清单」：哪些题/话术来自亲历，哪些缺口需要联网补。

### 第三步：联网搜索补缺（必须联网，但目标是补 reviews/ 没覆盖的部分）

针对第二步留下的缺口，发起搜索（至少 4 组）：

1. `"[公司名] 前端 面经 2025 2026"` — 近期真实面试题
2. `"[公司名] 面试题 [岗位方向]"` — 该方向高频题
3. 二面及以上：`"[公司名] [轮次] 项目深挖 面经"`
4. `"[公司名] 面试风格 流程"` — 流程与风格
5. AI/Agent 方向：`"[公司名] AI Coding Agent 面试 2026"`
6. `"[公司名] [岗位] 薪资"` — 薪资区间

从结果提取：面试流程、面试风格、高频题、偏好画像、薪资区间。**与第二步的亲历信息冲突时，以亲历为准**。

### 第四步：JD 深度分析

输出 **JD→经验映射表**：

| JD 要求 | 候选人对应经验 | 匹配度 | 话术要点 |
|---------|-------------|--------|---------|
| ... | ... | ★★★★★ | ... |

**匹配度评分**：
- ★★★★★：有 1:1 对应项目经验，可直接讲
- ★★★★：有相关经验，需桥接话术
- ★★★：间接相关，需拆解底层能力对应
- ★★：有了解但无深度实战
- ★：完全缺失，需兜底话术

**核心结论**：3 个最强匹配点 + 1-2 个需准备的薄弱点。
**结合第二步**：把候选人在同类面试中已暴露的薄弱点也并入这里，优先补强。

### 第五步：生成定向准备文档

保存到 `learning/interview/定向/[公司名].md`，必须包含：

#### 一、JD 分析与匹配度
- 1.1 JD 原文要点（职责 + 任职要求）
- 1.2 JD→经验映射表（第四步输出）
- 1.3 面试风格特点（维度/特点/应对策略）
- 1.4 面试流程（轮次/面试官/时长/核心内容/特点）

#### 二、我的亲历真题命中 ⭐（来自 reviews/）
单独成章，把第二步从本地面经库挖到的内容前置展示：
- 该公司亲历真题（如有）：原题 + 当时表现 + 本次改进话术
- 同类公司高频题：题目 + 出现过的公司 + 标准答法
- 候选人系统性薄弱点：必须在面试前补强的清单
- **3 致命硬伤专项**（13 场面经归纳，每次必过一遍）：①数据自证不了——每个数字备口径（5h→2h 是 5 需求对照、单环节提效、全链路 20-30%；85% 是文件列表重跑一致性，Readiness 82/83/62 评单次质量，互补不互替）；②AI 原理说不清——图片传参/Function Calling/Skills 触发/LangGraph 各背一句完整链路（别把倒排+BM25 说成 RAG）；③端到端闭环——不许用"老板让先做前两步"回避，给出执行→Review→自修复的设计认知
- **4 王牌素材**（每场至少抛一个）：准确率五维定义、负向搜索（搜不到也是证据）、消融实验（去 IR 层 82→54）、确定性 vs 非确定性分工

#### 三、基础高频题
按面经（亲历优先，联网补充）实际频率排序，分组：JavaScript 核心 / CSS / React·Vue / 浏览器与网络。每题标注：

- **回答要点** + **频率**
- **来源分类**（三档，决定准备优先级）：
  - `✅ 真题参考`：reviews/ 亲历原题——最高优先级，必须练到脱口而出
  - `🔗 相似题迁移`：跨公司/跨方向出现过的同类题——准备标准答法即可
  - `🧩 JD 推演题`：从 JD 反推但无面经佐证——准备思路框架，不必死记
- **答案分级**（自检尺子，每题必给）：
  - 🟢 好答案：戳中考察点 + 有数据/例子 + 结构清晰
  - 🟡 普通答案：正确但缺细节，听起来像背的
  - 🔴 危险答案：常见误区/会踩的雷（标出来才知道避开）

#### 四、手撕代码
该公司有手撕传统则分梯队列出（第一梯队附完整代码，其余附核心思路）+ 练习优先级；无则简略带过。

#### 五、算法题
基于面经提取常考方向与具体题目，标注难度 + 核心思路。

#### 六、项目深挖 + 场景设计
- 6.1 该公司追问模式（N 层递进表格）
- 6.2 项目话术锚点（对准 JD 关键词，技术细节引用 `archive/retrospectives/` 对应复盘）
- 6.3 场景设计题（基于 JD 业务场景出题）
- 6.4 AI Coding 方向额外含 AI 工程深挖题
- 6.5 **素材缺口清单**：本次面试前还需补的真实素材，每条标注「缺口类型 + 紧急度（P0/P1/P2）+ 获取来源」：
  - 需确认的数据/口径（如某项目具体转化率、提效百分比的来源）
  - 需补的项目细节（JD 要求但简历/复盘里没写的）
  - 需准备的案例（对应第二章薄弱点 + 3 致命硬伤专项）

#### 七、方向专项题（按 JD 自动选择）
- **AI Coding / SDD**：SDD 全流程、Vibe Coding vs 标准化研发、Harness 约束、Agent 框架、RAG、MCP 协议、Prompt Engineering、AI 代码质量管控；**3 硬伤高频追问**：你怎么保证准确率 / 图片怎么传给模型 / 你的工作流和直接调 Claude Code 区别 / Agent 死循环怎么防
- **传统前端**：性能优化、工程化、组件库设计、SSR、微前端
- **全栈**：BFF 设计、Node.js 中间件、数据库基础、CI/CD

#### 八、软素质 + HR 面
- 8.1 "为什么跳槽"（感恩过去 + 明确诉求 + 匹配该公司）
- 8.2 职业规划（分时间段话术）
- 8.3 该公司特有问题（基于面经和公司文化）
- 8.4 反问环节（2-3 个，必须与 JD 业务相关）
- 8.5 HR 面注意事项

#### 九、备战优先级

| 优先级 | 内容 | 时间 |
|--------|------|------|
| P0 | ... | 今天 |
| P1 | ... | 明天 |

#### 十、30 秒自我介绍（该公司定制版）
基于 JD 核心关键词 + 候选人最强匹配点。

#### 十一、面试当天一页纸
代码块输出：主线锚点 / 手撕必会（如考）/ 八股核心 5-8 关键词 / 项目追问 N 层防御 / 兜底句 3 句。

#### 十三、来源索引
列出本地引用（reviews/ 文件名）+ 联网来源 URL 与概要。

### 第六步：收尾与闭环

1. 告诉用户文档路径，提示可配合 `/模拟面试` 使用。
2. **闭环提示**：面试结束后，把真实问答补充到 `reviews/qa/[公司名].md`，下次准备同类公司时本 skill 会自动复用——面经库越用越强。

## 候选人背景

以实际读取的 `learning/interview/resume/v3.md` 为准，重点关注：当前定位、核心项目、技能栏、量化数据。

## 硬性原则

- **亲历真题优先**：第二步必须先挖 `reviews/`，与联网结果冲突时以亲历为准。
- **必须联网补缺**：禁止用旧数据或编造面经；联网搜索至少 4 组关键词。
- 匹配度分析必须诚实，缺失能力标 ★ 并给兜底话术；把候选人已暴露的系统性薄弱点纳入重点。
- 自我介绍和话术必须锚定 JD 关键词，不泛泛而谈。
- 手撕/算法题必须附完整代码或核心思路；场景题必须基于 JD 业务场景。
- 面试风格分析基于真实面经（亲历 + 联网），不靠猜测。
- 已知文化特点（如拼多多高强度、字节 OKR）在 HR 面部分标注。
- **题目来源三分类**：所有题目必须用 `✅ 真题参考 / 🔗 相似题迁移 / 🧩 JD 推演题` 标注，让候选人一眼看出哪些必须死磕、哪些准备思路即可。
- **答案必须有分级尺子**：高频题答案不能只给"话术要点"，必须附 🟢/🟡/🔴 三档对比，让候选人自检到哪一档。
- **素材缺口必须显式列出**：第六章 6.5 不能省——面试前必须有明确的"补什么"清单，而不是泛泛的"加强准备"。
