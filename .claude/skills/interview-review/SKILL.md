# /面试复盘

面试结束后使用。自动诊断失分点，生成改进方案，并回流新知识点到手机速查表。

## 用法

```
/面试复盘 星河觉醒 一面
```

然后粘贴录音转写文本。也可以一句话全给：

```
/面试复盘 星河觉醒 一面
[粘贴转写文本]
```

## 执行流程

### 第一步：清洗录音转写

1. **识别说话人**：区分面试官和候选人（候选人发言通常包含项目细节和技术术语）
2. **纠正ASR错误**：常见映射：
   - AR → AI, 四编码/SSC码 → Schema, PSG → PRD
   - note → Node.js, 壁府/地F → BFF, 衣服/VIVO → eval
   - PI pd/PI ID → PRD, 歪理 → Value, 模板琴 → 模板引擎
   - roll2 → Rollup, 仿修 → 返工, 跑领领008 → 跑了0-0-8
3. **提取Q&A对**：整理成结构化问答

输出清洗后的 Q&A 对列表。等用户确认后继续。

### 第二步：逐题深度分析

对每一道题输出：

**Q[N]: [问题]**
- 表面问的是：X
- 实际考察的是：Y
- 答到的要点：列出来
- 漏掉的关键点：列出来

| 维度 | 分数(1-10) | 问题 |
|------|-----------|------|
| 知识完整度 | ? | 漏了什么 |
| 表达清晰度 | ? | 哪里冗余/绕弯/口头禅多 |
| 追问防御力 | ? | 哪些追问没防住 |

**改进版答案**：
- 核心句（20字以内）
- 口语展开（3-5句）

**保存复盘产物**：逐题分析完成后，**立即**将所有改进版答案写入 `interview-experience/Q&A-list/[公司名].md`，不要延迟到后续步骤。格式：

```markdown
# [公司名] [轮次] — 面试优化答案

> 日期：YYYY-MM-DD | 通过概率：XX%

---

## Q1: [面试官问题]
**核心句**：[20字以内]
**展开**：[3-5句口语化回答]

## Q2: ...
```

如果 `interview-experience/Q&A-list/` 目录不存在就创建。

### 第三步：整体评估

| 项目 | 结果 |
|------|------|
| 通过概率 | XX%（理由） |
| 最大加分点 | |
| 最大失分点 | |
| 面试官风格判断 | |

**表达诊断**：
- 口头禅统计（"然后""就是""对吧"出现几次）
- 回答平均长度（太长/太短）
- 结构性（是否有开头-展开-收尾）

### 第四步：知识回流

**这是最重要的一步。** 分析面试中暴露但现有知识库没覆盖的知识点，生成 stealth.html 卡片。

**结合源码分析**：如果面试涉及候选人自己的项目（如 prd-tools、Schema中台等），必须先读取对应项目源码（如 `/Users/didi/work/prd-to-code`），基于实际实现生成准确的知识卡片，不能凭记忆或面试时的模糊回答来写。源码路径：
- prd-tools AI工作流 → `/Users/didi/work/prd-to-code`
- Schema中台/权益组件库 → `/Users/didi/work/genos` 或相关项目目录

判断是否需要回流：
- 某题卡壳，且原因是缺少该知识点 → 回流
- 面试官问了没准备过的方向 → 回流
- 回答不够准确需要纠正 → 回流
- 涉及自己项目的架构/机制但面试时讲不清楚 → 必须读源码后再回流

先读取 stealth.html 确认当前卡片状况，避免重复。然后为每个回流点生成卡片HTML：

```html
<div class="card" data-c="[分类]" data-kw="[关键词1] [关键词2]">
  <div class="card-head">
    <span class="tag [分类]">[标签]</span>
    <div class="card-title">[面试题/知识点]</div>
  </div>
  <div class="core">[核心答案，20字以内]</div>
  <div class="card-body">
    <p>[展开说明]</p>
    <p><strong>[关键点]</strong>[详细解释]</p>
  </div>
</div>
```

分类规则（data-c）：
- React/浏览器/TS/CSS基础 → `bagu`，标签 `#N`
- 项目设计/架构/工程化 → `proj`，标签 `A N`
- AI/LLM/Agent/RAG → `ai`，标签 `AI`
- 手写代码/算法 → `code`，标签 `手撕`
- 面试策略/话术/反问 → `emer`，标签 `软技能` 或 `反问`

输出：
1. 先列出所有待回流的知识点，等用户确认
2. 确认后输出完整卡片HTML
3. 告诉用户应该插入 stealth.html 的哪个位置（行号范围）
4. 用户确认后执行插入，然后 bump sw.js 的 CACHE_NAME

### 第五步：改进计划

**今晚必做3件事**：每件说清补什么 → 怎么补 → 补到什么程度

**下次面试注意**：2-3条针对性建议

### 第六步：自动提交推送（不询问用户）

复盘流程结束后，自动执行以下操作，**不需要等用户说"提交"**：

1. `git add` 所有变更文件（录音转写、Q&A答案、stealth.html、sw.js）
2. `git commit`，消息格式：`fix: [公司名][轮次]复盘 — [新增卡片摘要] + Q&A答案`
3. `git push` 到远程 main 分支
4. 输出 commit hash 和推送结果

## 候选人背景

- Schema驱动营销中台（BFF模板引擎 + 前端渲染引擎 + 联动系统）

## 硬性原则

- 基于事实分析，表现差直说差在哪，不安慰
- 不编造用户没说过的回答
- 知识回流只生成真正缺失的卡片，不重复已有知识点
- 每张卡片的核心答案必须能在手机上3秒内读完
- 改进动作具体到"今天花30分钟做xxx"，不要笼统说"加强练习"
- 插入 stealth.html 后必须 bump sw.js 的 CACHE_NAME（kb-vN → kb-vN+1）

## 卡片内容质量规则（防止"写得好但面试念不出来"）

### 1. 聚合不拆散

**同一面试题的知识点必须聚合到一张卡片里，不要拆成多张。**

反面案例：面试官问"介绍你的AI工作流"，答案散在5张卡片（定位/双插件/证据链/质量门控/评分），面试时翻不过来。

正面案例：一张"AI工作流完整介绍"卡片覆盖：是什么→为什么→怎么做（三大支柱）→效果→边界。

**规则**：如果回答某个面试题需要翻2张以上的卡片，说明需要合并。细节卡片保留给追问环节。

### 2. 可朗读性检查

**每张卡片的内容必须能直接念出来。禁止以下写法：**

| 禁止 | 原因 | 替代 |
|------|------|------|
| 缩写不展开（BF、SSC、PI ID） | 念不出来 | 首次出现写全称 |
| 箭头流无解释（A→B→C） | 口头说不清 | 每步用完整句子 |
| 括号压缩10个东西 | 口头报不完 | 拆成独立行或只列3个关键的 |
| 层级编号无上下文（L1/L2/L3） | 不知道在说什么 | 写明"第一层XX、第二层YY" |
| 代码/文件名当正文 | 面试不说文件名 | 用自然语言描述 |

### 3. core 字段 = 面试第一句话

`core` 字段必须是面试时脱口而出的**完整句子**，不是一个短语。

| 错误 | 正确 |
|------|------|
| `SSOT+证据链+多层门控保质量` | `把PRD自动蒸馏成结构化开发计划，靠SSOT+证据链+多层门控保质量` |
| `目录千差万别，按能力面描述` | `目录结构千差万别，但研发协作的能力面是稳定的——按能力面描述` |
| `WASM + ONNX Runtime 跑小型模型` | `能，WASM + ONNX Runtime 跑小型模型` |

### 4. 质量自检清单

生成每张卡片后，逐项检查：
- [ ] core 是完整句子，能直接念出来？
- [ ] card-body 每段是自然语言，不是缩写/代码/文件名列表？
- [ ] 面试官问这个问题，翻这一张卡就能答完，不用再翻第二张？
- [ ] 跟 Q&A 文件中的优化答案对比，卡片内容是否更差？如果是，用 Q&A 答案替代。
