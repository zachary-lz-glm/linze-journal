# /面试演练

面试前一键深度演练模式。AI 同时演「互联网大厂深度面试官」和「拿着速查页的模拟面试者」，演示每道题怎么翻卡念卡怎么改，最后出实战准备报告。**核心创新：演示真实翻卡路径**——不只给策略，演给你看怎么用 stealth.html 当弹药库。

## 用法

```
/面试演练 快手 电商 二面
```

然后粘贴 JD。也可以一次全给：

```
/面试演练 快手 电商 二面
JD：[粘贴JD全文]
```

如果是 AI Coding 实操环节专场：

```
/面试演练 快手 电商 二面 --实操专场
JD：[粘贴JD全文]
```

## 与其他 skill 的关系

| Skill | 模式 | 输出 | 适合场景 |
|-------|------|------|---------|
| `/面试准备` (interview-prep) | 静态准备 | 策略文档（13 章广度：HR/薪资/反问/手撕等） | 首次准备、广度覆盖 |
| `/模拟面试` (mock-interview) | 互动一问一答 | 实时评分 | 练表达节奏 |
| **`/面试演练` (interview-drill)** | **AI 自演完整轮** | **演练报告（深度演示）** | **临阵磨刀、看实战范例** |

**推荐组合**：`/面试准备`（出策略）→ `/面试演练`（深度演示）→ `/模拟面试`（练节奏）→ 面试 → `/面试复盘`（回流）。

## 执行流程

### 第一步：信息收集（必须并行）

| 资产 | 路径 | 用途 |
|------|------|------|
| 简历 | `learning/interview/resume/resume.typ` | 候选人最新背景 |
| 速查页 | `learning/interview-tools/stealth.html` | **演练的单一信源**（答案必须能映射到这里的卡） |
| 亲历面经 Q&A | `reviews/qa/*.md` | 第一手真题，优先于联网 |
| 亲历面经转写 | `reviews/transcripts/*` | 卡壳点和面试官风格 |
| 已有定向文档 | `learning/interview/定向/*.md` | 同类公司准备成果 |
| 项目复盘 | `archive/retrospectives/*.md` | 项目深挖弹药 |

**并行操作**：
1. 读简历（提取核心项目 + 量化数据）
2. 扫速查页（grep 所有 `card-title` 和 `data-kw`，建立"关键词 → 卡片"反向索引）
3. 挖 reviews/ 找该公司或同方向公司的亲历真题
4. 联网搜（至少 5 组）：
   - `"[公司名] 前端 面经 2025 2026"`
   - `"[公司名] AI Coding 实操 面试"`
   - `"[公司名] [轮次] 项目深挖"`
   - `"[公司名] 面试风格 流程"`
   - 通用补强：`"大厂 前端 AI Coding 实操 题目 2026"`

### 第二步：JD 关键词 → 速查页卡片映射

提取 JD 的：
- 3-5 个核心技术关键词（决定八股重点）
- 1-2 个业务场景关键词（决定项目话术锚点）
- 团队定位（业务/基建/AI）
- **是否含 AI Coding 实操环节**（决定要不要重点演示实操剧本）

输出 **JD→速查页卡片映射表**（这张表本身就是明天手机滑一遍的搜索速查）：

| JD 关键词 | 对应速查页卡片 | 搜索关键词 |
|-----------|-------------|-----------|
| 例：Schema 驱动 | A2 Schema 驱动架构 + A7 联动引擎 | `Schema 驱动` `联动引擎` |
| 例：AI Coding 实操 | 实操 7 步剧本 + 4 题型 prompt 模板 | `AI Coding 实操` `prompt 模板` |

### 第三步：设计 5 轮演练（基于追穿点矩阵 + 公司风格）

**5 轮标准模板**（可根据公司风格调整侧重点）：

| 轮次 | 主题 | 戳穿点 | 必须能映射的速查页卡片 |
|------|------|--------|---------------------|
| 1 | 主项目链路 + 数据口径 | 数字怎么算/对照组/分母 | 简历项目 1 + 速查页 A 系列 + 数据速查卡 |
| 2 | 架构权衡深挖 | 选型 trade-off/边界感 | 项目追问卡 + 防御卡 |
| 3 | AI 边界感（实操前置验证） | Vibe vs 标准 / Agent vs Workflow | AI 工程卡 + 实操 7 步剧本 |
| 4 | 事故 + 落地规模诚实度 | 真在用吗/规模指标 | 项目追问"线上事故" + 数据速查 |
| 5 | 公司定制（按风格） | 字节=算法 / 阿里=场景 / 腾讯=代码质量 / 美团=AI 机考 / 快手=实操预热 | 按 JD 业务场景出题 |

**轮次设计原则**：
- 每轮必须能映射到速查页 1-2 张具体卡（找不到卡的题改设计）
- 每轮埋 1-2 个真实追穿点（数据口径/AI 原理链路/落地规模）
- 至少 1 轮要设计"卡壳场景"，演示 5 条避坑话术怎么救场
- 总轮数严控 5 轮（多了用户记不住）

### 第四步：AI 自演完整轮（核心步骤）

每轮严格按以下结构输出：

```
## 🎬 第 N 轮：[主题]（戳穿点：[具体点]）

**🎯 面试官**：[问题全文，互联网大厂深度风格]

**📱 模拟面试者翻卡**：手机 stealth.html 搜 `[关键词1]` → 翻到「[卡片名]」
                      + 搜 `[关键词2]` → 翻到「[卡片名]」

**💬 答全文**：[基于卡片内容的念稿级答案，包含钩子+主线+追问应对+落地]

**🔍 面试官追问**：[戳穿点追问，必须真戳不软]

**📱 再翻卡**：[补强的卡片，给具体关键词]

**💬 再答**：[补充答案，控制在 3-5 句]

**⭐ 评判**：[通过/扣分点 + 加分点，必须诚实]
```

**自演硬性规则**：
1. **翻卡路径必须具体**——给"搜什么关键词 → 翻哪张卡"，不能模糊说"翻速查页找"
2. **答案必须基于速查页原文**——不能编新答案；找不到卡的题就老实承认"这道我没准备过，临场靠 7 步剧本走"
3. **追问必须真实戳穿**——不能为了让面试者好看就问软题
4. **评判必须诚实**——低分轮次要明确扣分，不能都给通过
5. **偶尔设计卡壳**——演示"边收边审"或"避坑话术"的救场

### 第五步：整体评估

输出 5 维评分表：

| 维度 | 评分(1-10) | 说明 |
|------|-----------|------|
| 项目深度 | ? | 5 个项目追穿点能接住几个 |
| 数据真实性 | ? | 口径背得硬不硬 |
| AI 边界感 | ? | Vibe vs 标准、实操前置验证 |
| 架构权衡 | ? | 选型 trade-off |
| 事故/诚实度 | ? | 不甩锅、不夸大 |

加 **通过率**（百分比）+ **3 个最可能挂的方向**（基于演练暴露的真实薄弱点）。

### 第六步：生成报告

保存到 `learning/interview/定向/[公司名]-演练.md`，章节结构：

#### 一、JD 核心关键词 + 速查页对应卡清单
（第二步的映射表，明天手机可直接查）

#### 二、5 轮演练完整记录
（第四步的全部内容，含翻卡路径）

#### 三、整体评估表 + 通过率 + 风险预判
（第五步的输出）

#### 四、追穿点专项防御
（基于本次演练暴露的薄弱点，给针对性补强话术）

#### 五、明天实操环节特别提示（如 JD 含 AI Coding 实操）
- 实操 7 步剧本速记（复述→喂上下文→分步→边收边审→跑验证→加边界→讲取舍）
- 速查页搜索路径速查表（实操 4 题型 → prompt 模板）
- 5 条避坑话术索引

#### 六、临场一页纸（手机滑一遍就能上战场）
- 主线锚点（3 个最强匹配点）
- 速查页搜索关键词速查（10 个高频词 → 卡片名）
- 兜底句 3 句（被问倒时的救场）
- 应急话术（被戳穿数据/AI 原理/落地规模时的标准应对）

### 第七步：闭环

1. 告诉用户报告路径
2. 提示配合 `/模拟面试` 练节奏，面试后用 `/面试复盘` 回流真实问答

## 候选人背景

以实际读取的 `learning/interview/resume/resume.typ` 为准。重点关注：
- AI 工程方向（SDD / prd-tools / prd2code-gen）
- 中台架构（Schema 驱动 / 双重求值联动 / 权益组件库）
- 量化数据（5h→2h / 80% / 40%→85% / 400+ SKU / 20+ 人推广）
- 2 年 A 绩效 / 4 年大厂经验

## 硬性原则

- **速查页是单一信源**：所有答案必须能映射到 `stealth.html` 的某张卡，不能编新答案。
- **翻卡路径必须具体**：给"搜什么 → 翻哪张"，不能模糊。
- **评判必须诚实**：低分轮要承认扣分，不能为了好看都给通过。
- **联网必须做**：禁止用旧数据，至少 5 组搜索。
- **本地真实面经优先**：与联网冲突时以 `reviews/` 为准。
- **报告必须可执行**：临场一页纸必须手机滑一遍能用，不能是抽象策略。
- **轮数严控 5 轮**：多了用户记不住。
- **每轮必须含翻卡动作**：这是本 skill 区别于 mock-interview 的核心。

## 互联网大厂前端 AI Coding 面经速查（出题参考）

| 公司 | 面试风格 | AI Coding 考察特点 | 第 5 轮定制建议 |
|------|---------|------------------|---------------|
| 字节跳动 | 算法优先、拼手速 | 默认你具备 AI 辅助下的极速编码，重点考逻辑纠错 | 给含 Bug 的 AI 生成代码让定位 |
| 阿里 | 场景导向、业务落地 | 重业务规则理解（如签到积分题） | 基于 JD 业务场景出设计题 |
| 腾讯 | 追问项目深度、严控代码质量 | 命名/注释/规范扣分细 | 让现场 review 一段代码 |
| 美团 | 场景设计、AI 机考 | 已有机考（如商品自动审核系统） | 给完整场景题让 30 分钟出原型 |
| 快手 | 内部 AI Coding 实践成熟 | 实操环节 + 录屏 | 重点演示 7 步剧本 |
| 百度 | AI 技术深度 | 注重原理 | 追问模型/Token/上下文管理 |
| 拼多多 | 高强度、节奏快 | 基础+手撕为主 | 高频基础题连击 |

## 评分维度参考（互联网大厂通用）

| 维度 | 权重 | 说明 |
|------|------|------|
| 内容相关性 | ⭐⭐⭐ | 答案是否切题、功能是否完整 |
| 逻辑结构 | ⭐⭐⭐ | 代码/答案逻辑是否清晰 |
| 专业深度 | ⭐⭐ | 技术选型合理性、性能考量 |
| 表达沟通 | ⭐⭐ | 边写边讲（Think Out Loud） |
| **AI 工具使用能力** | ⭐⭐⭐ | Prompt 质量、是否高效利用 AI |
| 代码规范 | ⭐⭐ | 命名、注释、错误处理 |
| Debug 能力 | ⭐⭐ | 发现并修复问题的速度 |

> 关键转变：评分重点从"能否手写"转向"能否高效利用 AI 并验证其输出正确性"。本 skill 演练时重点训练这个能力。
