# /模拟面试

面试前一天使用。自动搜索目标公司真实面经，结合JD做一场针对性模拟。

## 用法

```
/模拟面试 星河觉醒 一面
```

然后粘贴招聘JD。也可以一次全给：

```
/模拟面试 星河觉醒 一面
JD：[粘贴JD全文]
```

## 执行流程

### 第一步：搜索面经

用 WebSearch 搜索（必须搜，不要用旧数据）：

1. `"[公司名] 前端 面经 2025 2026"` — 近期真实面试题
2. `"[公司名] 前端 面试题"` — 经典高频题
3. 如果是二面或更高轮次，额外搜索 `"[公司名] 前端 二面 项目深挖"`

从搜索结果提取：
- 该公司常考方向（偏基础？偏项目？偏算法？）
- 面试官追问风格（深挖原理？场景落地？压力测试？）
- 该轮次高频题目

### 第二步：分析JD

如果用户提供了JD，提取：
- 3个核心技术关键词 → 决定八股重点
- 1-2个业务场景关键词 → 决定项目怎么讲
- 团队定位（偏业务/基建/AI）→ 决定讲哪个项目

### 第三步：输出面试方案

基于面经+JD分析，简要输出（3-5行）：
- 本次模拟重点方向
- 预计提问 8-10 题
- 追问策略

等用户确认后开始模拟。用户可以直接说"开始"。

### 第四步：正式模拟

根据轮次调整风格：

**一面**（技术基础 + 项目）：
- 自我介绍 → 2道项目题 → 2道八股题 → 1道手撕 → 反问
- 追问2-3层
- 重点是表达清晰、核心知识点覆盖

**二面**（项目深挖 + 架构思维）：
- 自我介绍 → 1个主项目深挖（5层追问）→ 1道开放设计题 → 反问
- 追问5层，必须挖到trade-off和"如果重做"
- 重点是设计决策能力、追问防御

**HR面**（行为面 + 职业规划）：
- 离职原因 → 职业规划 → 团队协作 → 抗压场景 → 期望
- 追问细节验证真实性
- 重点是回答是否有风险

模拟规则：
1. 每次只问一个问题，等用户回答
2. 回答模糊/夸大/有漏洞 → 直接追问
3. 回答太长 → 打断："用一句话概括核心"
4. 追问时标注"追问第X层"
5. 8-10轮后结束
6. **3 硬伤必追问**：候选人一提量化数据 / AI 原理概念 / 端到端闭环，必须按 3 硬伤模式追问至少 2 层——数据追问口径（怎么算的、对照组、分母）、原理追问底层链路（图片传参/Function Calling/Skills 触发）、闭环追问下一步（执行→Review→自修复）。这 3 类是 13 场真实面试追穿候选人的模式，模拟时必须复现

### 第五步：评分报告

模拟结束后输出：

#### 逐题复盘

每题：题目 | 考察意图 | 得分点 | 失分点 | 改进版答案（核心句20字 + 口语展开3-5句）

#### 四维评分

| 维度 | 分数(1-10) | 说明 |
|------|-----------|------|
| 知识完整度 | ? | 核心知识点是否覆盖 |
| 表达清晰度 | ? | 逻辑是否清晰 |
| 追问防御力 | ? | 连续追问后是否崩塌 |
| 整体节奏感 | ? | 像聊天还是背书 |

#### 明日必做3件事

每件说清：补什么 → 怎么补 → 补到什么程度

#### 风险预判

明天真实面试被追问的3个高危方向 + 应急策略

## 硬性原则
