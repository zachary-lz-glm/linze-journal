# 2026-04-22 周二

## 📐 Reference 领域知识标准化 & PRD 蒸馏优化（技术调研）

### 调研背景
- mcp-bff-gen 当前 reference 是**单层（BFF）× 单项目（marketing-bff）**知识，无法扩展到前端/后端
- PRD 蒸馏准确率最大瓶颈是**PRD 输入质量**（影响 ★★★★★），而非蒸馏算法

### Part 1: 三层分离的领域知识架构

**核心设计：reference 分 4 个层级组织**

```
reference/
├── domain/            # 层无关的领域知识（共享）
│   ├── glossary.md    # 业务术语表
│   ├── campaign-types.md  # 活动类型枚举
│   ├── field-registry.md  # ⭐ 全量字段注册表（核心枢纽）
│   ├── business-rules.md  # 层无关业务规则
│   └── enums/
├── layers/            # 层特定实现知识
│   ├── bff/           # BFF 层（已有内容重构）
│   ├── frontend/      # 前端层（新增）
│   └── backend/       # 后端层（新增）
├── templates/         # 确定性模板（按层×活动类型矩阵）
└── cross-cutting/     # 跨层关注点
    ├── data-flow.md
    ├── field-lifecycle.md
    └── integration-points.md
```

**4 个核心原则**
1. **field-registry 为中心** — 定义每个字段的三层映射（BFF / Frontend / Backend），是连接 domain 和各层实现的枢纽
2. **领域知识层无关化** — `domain/` 不含实现细节，新增层时 domain 零修改
3. **模板按层×活动类型矩阵** — BFF×3种活动 / Frontend×3种 / Backend×3种
4. **知识构建流程标准化** — 4 阶段：Domain 采集 → Layer 采集 → 模板构建 → 持续维护

**知识质量度量**
- 覆盖率 > 95%（field-registry 字段数 / 实际字段数）
- 模板命中率 > 70%（高频场景）
- 准确率 > 98%（reference vs 代码实际行为）
- 蒸馏成功率 > 80%（high confidence 字段占比）

### Part 2: PRD 蒸馏优化

**蒸馏效果 5 大依赖因素**

| # | 因素 | 影响程度 |
|---|------|---------|
| 1 | PRD 输入结构 | ★★★★★ |
| 2 | Reference 覆盖度 | ★★★★☆ |
| 3 | 蒸馏 JSON Schema | ★★★☆☆ |
| 4 | 置信度校准 | ★★★☆☆ |
| 5 | 上下文窗口 | ★★☆☆☆ |

**4 个核心瓶颈**
1. PRD 术语不统一（"目标司机" vs "参与人群" vs "司机筛选条件" → 都是 target_crowd）
2. 字段归属判断模糊（跨模块描述时决策树失效）
3. 组件类型推断不可靠（"选择" = Select 还是 Radio？）
4. 验证规则隐含在自然语言中（LLM 容易遗漏）

**解决方案：AI 友好的 PRD 标准格式**
- Markdown 优先（LLM 解析效率最高）
- 表格化字段描述（结构化 > 自然语言）
- 分区对应 target_file（PRD 结构与 BFF 文件结构对齐）
- 标签化组件（`[Input]` / `[Select]` / `[DateRangePicker]`）
- 引用术语表（统一词汇 + domain glossary ID）

**蒸馏 JSON Schema 扩展**
- 新增 `layers` 维度（bff / frontend / backend）
- 新增 `cross_layer_deps` 跨层依赖描述
- 新增 `preview_placeholders`

### 收益预估

| 优化项 | 预期效果 |
|--------|---------|
| PRD 结构化 | 蒸馏 high confidence 40% → 80%+ |
| 术语统一 | 字段映射准确率 ~70% → ~95% |
| 组件标签化 | 组件推断错误减少 80% |
| 验证规则结构化 | 验证规则遗漏减少 90% |
| Backend Contract | 跨层数据契约一致性 100% |

### 优先级排序

| 优先级 | 事项 | 工作量 |
|--------|------|--------|
| P0 | PRD 标准化模板 + 术语表 | 1 周 |
| P0 | field-registry.md 骨架 | 1 周 |

### 5 个关键结论
1. **PRD 输入质量是蒸馏效果的最大杠杆** — 投入 PRD 标准化的 ROI 远高于优化蒸馏算法
2. **Reference 的核心是 field-registry** — 连接 domain 知识和各层实现的枢纽
3. **三层分离架构让扩展成本线性** — 新增层只需添加子目录，domain 零修改
4. **"给 AI 写 PRD"是一种新技能** — PM 需要学结构化表达，但也让 PRD 对人更清晰
5. **模板命中率决定效率上限** — 高频场景模板化后 70%+ 不需 LLM
