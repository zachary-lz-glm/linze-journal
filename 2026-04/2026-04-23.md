## 二、工具定位升级：Context Engineer

### 核心认知转变

```
V1: 代码生成器 → PRD JSON → 模板引擎 → 代码
V2: 流程化能力提供者 → 提供 reference 格式 + 构建引导
V3: Context Engineer → 为 vibe coding 准备高质量上下文（当前）
```

**引用 Anthropic Context Engineering**：

> "Find the smallest possible set of high-signal tokens that maximize the likelihood of some desired outcome."

工具本质不是代码生成器，而是**为 vibe coding 准备高质量上下文的 Context Engineer**。

- 开发者拿到产出（蒸馏+分析）后，可以在**任意 AI Agent**（Claude Code / Cursor / Copilot）中完成 vibe coding
- **代码生成是可选步骤，不是核心**
- Reference 是 AI Agent 的持久化上下文，任何 Agent 都可消费

### 对现有架构的影响

1. PRD 蒸馏产出不再是 JSON，而是 AI 友好的 markdown spec
2. `_bff-gen-output/` 下所有产出改为 .md 格式
3. 工作流从 6 步调整为 5 步（合并/简化）

## 四、00-index.md 契约模型

### 设计理念

00-index.md 从"导航索引"升级为**工具与 reference 的契约文件**。

新增"可用性标记"：

```markdown
## 可用文件
- ✅ 01-registry.json
- ✅ 02-topology.md
- ⬜ 06-anti-patterns.md   ← 未填充
- ✅ 07-mechanisms.md
```

### 优雅降级策略

| 文件状态 | 工具行为 |
|---------|---------|
| 存在且有内容 | 正常消费 |
| 存在但是空骨架 | 跳过，相关校验降级为 warning |
| 不存在 | 跳过，00-index 标记为不可用 |

**核心原则**：Reference 是"有最好，没有也能跑"。

### 对各步骤的影响

| 步骤 | 缺关键 reference 时的降级 |
|------|------------------------|
| PRD 蒸馏 | 缺 01→不校验枚举；缺 08→PRD 关键词映射靠 AI 猜 |
| 项目分析 | 缺 02→无法精确定位文件，准确率下降 |
| 变更计划 | 缺 04→AI 自由发挥代码结构 |
| 验证修正 | 缺 05+06→降级为纯编译检查 |

### 核心问题


## 八、行业研究与验证（延续自 04-22）

新增 2 个行业实践对标：

### Spec-Driven Development（GitHub Spec Kit）

> "The specification is the primary artifact, code is an implementation detail."

工作流：Specify → Plan → Tasks → Implement

**对 prd2code-gen 的启示**：
- PRD 蒸馏产出 = AI spec（Specify 阶段）
- 项目分析+变更计划 = Plan+Tasks
- 验证了产出应该是 developer-friendly spec，不是 machine-readable JSON

### PRD for AI Coding Agents（行业实践）

关键原则：
- PRD 应分阶段，每次 5-15 分钟 AI session 产出一个可执行任务
- "DO NOT CHANGE patterns"——AI 应遵循既有模式
- "Research before specification"——先了解现状再写规格

**对 prd2code-gen 的启示**：
- 蒸馏产出应该是多个小 story，不是一个大的 monolithic spec
- 与 BMAD epic→story 分解模型高度吻合
- 04-patterns + 06-anti-patterns 就是 "DO NOT CHANGE patterns" 的具体化

### 行业趋势总结

| 趋势 | prd2code-gen 对应 |
|------|------------------|
| Context Engineering | 00-index.md + 按需加载 |
| Spec-Driven Dev | 蒸馏产出 = AI spec |
| PRD for AI Agents | PRD → epics → stories |
| Progressive Disclosure | 9 文件分级加载 |
