## 二、BFF Schema 生成 — 模板驱动 + 步骤分发

### 数据流

```
GET /create/render?step_name=basic
    ↓ Controller → 调后端 API → 拿原始数据
    ↓
getRenderSchemaTemplate(step, ctx)
    ├── BASIC  → basic.ts       (~1200 行，20+ 字段)
    ├── GROUP  → group.ts       (目标人群)
    ├── RULES  → rules/         (最大最复杂，25 种活动类型子模板)
    ├── MESSAGE → message.ts    (推送/标题/描述 ~980 行)
    └── PREVIEW → preview.ts
    ↓
Builder.parseOnce(templateStuff, apiData)
    ├── 遍历 entries，执行 handler → 生成 Schema 节点
    ├── afterEach: mixinDefaultFields + mixinValue → 合并后端数据
    └── afterAll: 提取 content_schema 数组
    ↓
返回 { content_schema: [...], steps: [...], preview: [...] }
```

### 模板结构

```typescript
{
  hooks: { beforeEach, afterEach, afterAll },
  template: [
    {
      path: 'basic.content.campaign_type',   // API 数据取值路径
      dataIndex: 'campaign_type',             // 字段标识
      handler: (ctx) => ({                    // 生成 Schema 节点
        d_component: FormDComponentType.RADIO,
        type: RadioType.CARD,
        rule: [{ required: true }],
        label: { display: 'Campaign Type' },
        options: [...]
      })
    }
  ]
}
```

**关键认知**：BFF 不是手写 JSON，而是模板引擎驱动。handler 描述字段结构，hooks 自动合并后端数据（visible/disabled/options/value），Builder 编译输出 content_schema。


## 四、injectDActions — 双重求值联动机制（BFF 最精巧的设计）

联动配置通过 `injectDActions` 声明式注入，**一次声明、双重求值**：

```
injectDActions(schema, [actionConfig1, actionConfig2])
    ↓
actionConfig 内部:
  register(path) → 注册依赖，返回 $deps[N]
  fetch().post(url, params) → 声明 API 调用
  return state: { visible: "$deps[0]===32" } → 声明表达式
    ↓
双重求值:
  1. BFF 端: safeEval(当前值) → 设初始 visible/disabled (SSR 就是对的)
  2. 序列化 d_actions → 前端 listener 触发时再 eval
```

**两种联动类型**：

| 类型 | 机制 | 适用场景 | 示例 |
|------|------|---------|------|
| **fetchApi** | 前端调 BFF → 重生成 Schema → 替换字段 | 选项需从后端获取 | 城市变化 → 活动类型选项更新 |
| **state** | 前端 eval 表达式 → 更新显隐/禁用 | 纯条件判断 | 活动类型=推送 → 显示推送模板 |

```json
// fetchApi 型
{ "dependencies": ["basic.city"], "callback": {
  "fetchApi": "/bff/schema/campaignType", "params": { "city": "{{$deps[0]}}" } } }

// state 型
{ "dependencies": ["basic.campaign_type"], "callback": {
  "state": { "visible": "{{$deps[0]===32}}" } } }
```

## 六、前端联动执行完整链路

### 阶段一：注册监听（FieldWrapper mount）

```typescript
// FieldWrapper/index.tsx
useEffect(() => {
  if (d_actions) registerEvent(d_actions)
}, [d_actions])

// 对每个 dependency 注册 listener.on
actions.forEach(action => {
  action.dependencies.forEach(event => {
    listener.on(event, (params) =>
      parseCallback({ form, action, dataSet: params, dispatch })
    )
  })
})
```

### 阶段二：触发联动（onFormChange）

```typescript
// StepForm
const onFormChange = (changedValues) => {
  Object.keys(changedValues).forEach(k => {
    let stateKey = `${currentStep}.${k}       // 拍平路径
    listener.emit(stateKey, {
      ...dataSet,                             // 全量表单数据(Redux)
      [stateKey]: createForm.getFieldValue(k)  // 变更值
    })
  })
  dispatch(saveFormData(values))
}
```

### 阶段三：parseCallback 分发

```
parseCallback({ action, dataSet, form, dispatch, listener })
    ↓
┌── callback.state ─────────────────────────────────────┐
│ state.value?                                           │
│   ├── "clear" → 清空                                   │
│   ├── 含 "{{...}}" → eval(schemaExpression(value))     │
│   └── 字面量 → 直接设值                                 │
│   → listener.emit → 支持链式联动 a→b→c→d               │
│                                                        │
│ state.visible/disabled/label/type/options?             │
│   → eval(schemaExpression(expression))                 │
│   → dispatch(updateItemSchema) → 字段重渲染             │
│                                                        │
│ state.attrs? → 同上模式，逐个 attr eval                 │
├────────────────────────────────────────────────────────┤
│ callback.fetchApi                                      │
│   → eval(schemaExpression(param)) 解析参数             │
│   → dajaxRequest(method, fetchApi, params)             │
│   → res.data 直接 merge 到字段 Schema                  │
│   → dispatch(updateItemSchema) → 字段重渲染             │
│   → listener.emit → 链式联动                           │
└────────────────────────────────────────────────────────┘
```

### 阶段四：schemaExpression 表达式解析

```typescript
// tools.ts
export const schemaExpression = (str: string) => {
  return String(str.match(/(?<={{).*?(?=}})/))
    .replace(/\$deps\[(\d+)\]/g, (_, i) => `dataSet[deps[${i}]]`)
    .replace(/\$\[([^\]]+)\]/g, (_, path) => `dataSet['${path}']`)
    .replaceAll('$', '')
}
```

**解析示例**：

```
输入: "{{$deps[0]===32}}"
  ↓ 提取 {{...}}
"$deps[0]===32"
  ↓ $deps[0] → dataSet[deps[0]]
"dataSet[deps[0]]===32"
  ↓ eval() — deps[0]="basic.campaign_type", dataSet["basic.campaign_type"]=1
  ↓ 1===32 → false
```

**链式联动**：parseCallback 执行完 state.value 或 fetchApi 响应后，再次 `listener.emit`，支持 a→b→c→d 级联联动。

## 七、完整链路总图

```
BFF injectDActions 生成 d_actions JSON
    ↓ SSR safeEval 求初始 visible/disabled
    ↓ 序列化到 Schema 返回前端
前端 FieldWrapper mount → listener.on(dependency, parseCallback)
    ↓
用户改字段 → onFormChange → listener.emit(formatName, dataSet)
    ↓
parseCallback 分发
    ├── state 型: schemaExpression → eval → updateItemSchema → 重渲染
    │   └── 再 emit → 链式联动
    └── fetchApi 型: eval params → API → BFF 重生成 → updateItemSchema → 重渲染
        └── 再 emit → 链式联动
```

## 八、多场景数据提取适配

同一字段在不同请求场景下路径不同：

```
创建渲染: apiData.basic.content.campaign_type.value
保存下一步: apiData.values.basic.campaign_type
```

`getAnyDataGadget.ts` 按优先级尝试所有路径，`injectShadowValue` 用 `__fieldName__` 双下划线注入最新值覆盖后端旧值。

## 九、全链路 STAR 讲稿

**S**: 23 种活动类型，前后端各写一遍，3-5 天/类型，联动散落易漏改。

**T**: BFF 自动生成 Schema，前端直接渲染执行联动，新增活动类型前端零代码。

**A**: 三层设计：
- **BFF Schema 生成**：模板引擎驱动，handler 生成 Schema 节点，hooks 合并后端数据。injectDActions 一次声明双重求值——BFF safeEval 设初始状态 + 序列化 d_actions 给前端联动。联动分 fetchApi 型（回调 BFF 重生成）和 state 型（前端 eval）。
- **前端渲染**：content_schema 遍历渲染，d_component 双轨映射（switch-case + RegisteredComponents 35+），formatName 拍平嵌套路径做 Redux key + listener 事件名。
- **联动执行**：listener 发布订阅，parseCallback 分发 fetchApi/state 两型，支持链式联动 a→b→c→d。

**R**: 前端零代码，3-5 天→1-2 天，23 种共享一套引擎，SSR 无闪烁。
