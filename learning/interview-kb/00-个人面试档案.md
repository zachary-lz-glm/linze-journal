## 2. 三条核心项目线（面试主轴，按重要性）

### 线一：Schema 驱动营销中台（滴滴 · 主项目）
- **一句话**：BFF 模板引擎生成 Schema + 前端渲染引擎动态消费，运营配 Schema 就能上新活动，前端零代码、不发版。
- **核心机制**：injectDActions（一次声明、双重求值——BFF 算初始状态，前端运行时再算联动）；联动本质是**字段有向图依赖**，引擎底层解决依赖图建模 + 拓扑排序求值 + 循环检测。
- **成果**：新增活动上线 2 周→几小时；一套引擎服务活动表单 + 权益 SDK 两个业务，维护成本降一半。

### 线二：权益组件库 benefit SDK
- **一句话**：Schema 渲染引擎的第二个消费者，Register 注册模式接入，30+ 组件靠配置覆盖国际化多国家多业务线。
- **核心机制**：SDK 内置通用组件 + 业务方运行时 register 定制组件（不用 fork 改源码）；组件靠**配置组合**覆盖国家/业务线差异，不是堆组件数量。
- **工程化**：Rollup 双模（ESM + CJS + d.ts）+ peerDependencies 声明 React + Changesets 管版本 + yalc 本地验证。

### 线三：AI 工程工作流 prd-tools（差异化亮点，AI 方向必讲）
- **定位**：填「AI 怎么理解已有项目」这个空白领域（Cursor/Copilot 解决从 0 到 1 写新代码，不管已有几万行历史项目）。
- **双轨**：
  - 团队推广版（生产在用）：PRD→Plan 蒸馏，11 步流水线，已推广到 20+ 人团队。
  - 端到端原型 prd2code-gen（个人探索）：PRD→Code 完整 6 步闭环（双路径生成 + 自验证自修复 + eval 反哺）。
- **核心创新**：证据链防幻觉（每条结论挂证据 ID）+ Reference 六文件知识结构 + 多层门控（五维 quality-gate + Spec 硬停止 + 独立 agent 终检）。
- **成果**：PRD→Plan 从 5h/需求降到 2h/需求。

## 5. 两个 AI Agent 项目（被深挖 AI 时讲，详见 03-AI项目实战.md）

- **Manager Agent（多智能体编排总管，端口 13106）**：14 个专业 Agent 之上的编排层。LangGraph 38 节点状态机 + Bandit/RL/Causal 三套自适应路由 + 自我进化飞轮（shadow→金丝雀→A/B）+ HITL 人工确认 + 证据闸门。
- **Code Assist（工程化 Coding Agent，端口 13103）**：Cursor 类工程化 Coding Agent。第一性原理「模型 + 写权限 = 事故」，read-before-write **四层系统级强制**（工具白名单 → 三重门禁 → context 行精确匹配 → 路径白名单 + sha256 乐观锁 + 原子写）+ 双层沙箱 + Prompt A/B 自进化。
- **关系**：Manager 是总管（编排），Code Assist 是它调度的下游专家之一（工程化安全），两者上下游。
