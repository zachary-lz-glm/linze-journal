## 三、流式渲染：SSE vs WebSocket

| 维度 | SSE | WebSocket |
|---|---|---|
| **数据方向** | 单向 Server → Client | 双向全双工 |
| **最佳场景** | LLM token 流式输出 | 语音对话、协作编辑、实时画板 |
| **重连** | 内置自动重连（Last-Event-ID） | 需手动实现心跳+重连 |
| **协议开销** | 低（基于 HTTP/1.1） | 较高（独立 TCP + 握手） |
| **浏览器支持** | 原生 EventSource | 原生 WebSocket API |

**行业共识**：
- ChatGPT/Claude 类文本对话 → **SSE 足矣**，每 1-5 token 推送一次 chunk
- 语音/视频 AI → **必须 WebSocket**，双向实时音频流
- 复杂 AI 应用 → **两者组合**：SSE 流式输出 + WebSocket 状态同步

**关键数据**：SSE 与 WebSocket 的 framing overhead 差异（~1ms vs ~50ms）在数千 token 累积后用户可感知。生产环境中 SSE 的平均 TTFB（首 token 延迟）约 200-500ms。

## 四、Streaming UI 设计模式（2026 核心范式）

### Token-by-Token Rendering Pipeline

```
LLM Server → SSE Stream → Stream Parser → React State → DOM Update
     ↓            ↓            ↓              ↓            ↓
 生成 token   分帧传输    解析+缓冲     useState 更新   增量 DOM patch
```

### 五大 Streaming UI 模式

| 模式 | 说明 | 代表产品 |
|---|---|---|
| **Token Streaming** | 逐 token 渲染，最基本的 AI 文本流式模式 | ChatGPT |
| **Progressive UI** | 先渲染骨架/占位组件，数据到达后渐进填充 | Claude Artifacts |
| **Generative UI** | AI 不只生成文本，直接生成可交互的 UI 组件 | Vercel AI SDK |
| **Streaming Tool Calls** | 工具调用状态实时反馈（搜索中→读取中→生成中） | Perplexity |
| **Multi-modal Streaming** | 文本+图片+代码混合流式输出 | ChatGPT GPT-4o |

## 六、语音/视频 AI 交互

| SDK/API | 类型 | 延迟 | 关键特性 |
|---|---|---|---|
| **OpenAI Realtime API** | 全双工语音对话 | <300ms | WebSocket 端到端，无需 STT→LLM→TTS 串行 |
| **Deepgram** | STT/TTS | 超低延迟 | 流式语音识别，AI 交互首选 STT 引擎 |
| **Azure Voice Live API**（2026 Preview） | 双向语音 | WebSocket | 微软 2026 新发布的实时语音 API |
| **VideoSDK** | 视频 + AI Agent | 低延迟 | 开源 AI 语音代理框架 |
| **Agora React SDK** | 实时音视频 | 超低延迟 | 声网前端 React 封装 |
| **OpenAI Agents SDK** | 语音 Agent 框架 | — | 端到端语音 Agent 构建工具 |

**核心趋势**：从"录音→STT→LLM→TTS→播放"五步串行，进化为 **WebSocket 全双工实时流**——用户说话的同时 AI 就在处理和响应，延迟从秒级降到亚 300ms。

## 七、AI 产品前端架构设计

### 2026 主流架构分层

```
┌──────────────────────────────────────────────────────┐
│                    Presentation Layer                 │
│   React / Vue / Svelte + AI UI Components            │
│   (Token Rendering, Generative UI, Loading States)   │
├──────────┬───────────┬───────────┬────────────────────┤
│ AI SDK   │  State    │ Streaming │  Real-time         │
│ Layer    │  Manager  │ Engine    │  Transport         │
│(Vercel AI│(Zustand/  │(SSE parser│(WebSocket /        │
│ SDK 5)   │ Jotai)    │+ buffer)  │ WebRTC)            │
├──────────┴───────────┴───────────┴────────────────────┤
│              Adapter / Gateway Layer                   │
│     (API Routes → LLM Provider APIs + Tool Calls)     │
├───────────────────────────────────────────────────────┤
│              AI Backend Services                       │
│    (LLM / RAG / Agent / Memory / Tools / Vector DB)   │
└───────────────────────────────────────────────────────┘
```

### 五大核心设计模式

| 模式 | 说明 | 代表产品 |
|---|---|---|
| **Streaming-First** | 所有 AI 输出默认流式，优化 TTFB 和感知延迟 | ChatGPT, Claude, Perplexity |
| **Server Components + AI** | RSC 负责初始渲染，客户端负责流式更新 | Next.js AI 应用 |
| **Micro-Frontend for AI** | AI 功能独立微前端部署，与主应用解耦 | 企业级 AI 平台 |
| **Edge + AI** | AI 请求在边缘节点处理 | Vercel AI SDK / Cloudflare Workers AI |
| **Offline-First AI** | 本地小模型 + 云端大模型混合 | AFFiNE, ollama + web |

**新兴协议**：**AG-UI**（Agent-UI Protocol）——专门为 AI Agent 与前端通信设计的开放事件协议，解决 Agent 的"最后一公里"问题。

## 八、总结

### 格局
Coding Agents 是增长最猛的赛道（opencode +1,425/28d、Claude Code +783、Codex +735），SSE 是 AI 文本流式的事实标准，Vercel AI SDK 5 + AI Elements 是 2026 AI 前端首选工具链，语音 AI 进入全双工亚秒级时代。

### 前端工程师核心机会

| 方向 | 具体技能 | 市场需求 |
|---|---|---|
| **流式渲染工程** | SSE 解析、token 级增量 DOM、断线重连、性能优化 | 所有 AI 产品必须 |
| **AI 交互组件** | 对话气泡、代码块实时渲染、Markdown 流式、多模态内容 | Chat/Product 类 |
| **语音/视频 AI** | WebRTC + WebSocket 混合架构，实时音频可视化 | 新兴方向 |
| **AI 状态管理** | 对话上下文、工具调用状态、流式 UI 更新的状态同步 | 复杂 Agent 应用 |
| **Generative UI** | AI 动态生成可交互组件，从文本到 UI 的范式转换 | 前沿方向 |
