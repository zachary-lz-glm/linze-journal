### Schema 驱动营销中台 — BFF 动态生成 + 前端动态渲染

*前端架构 + BFF 架构设计 · React · TypeScript · Node.js · Serverless · Lerna + Nx*

**背景**：国际化营销中台覆盖 10+ 国家、20+ 种活动类型，每种活动有不同字段组合和联动规则。传统方式每新增一种活动需要前后端各写一套页面和逻辑，开发周期长、容易出错。

**行动**：
- BFF 模板引擎：模板驱动 Schema 生成，字段按模板分布式管理，联动模板从渲染模板提取 handler 复用
- 双重求值联动：一次声明联动配置，BFF 端求初始状态（SSR 正确）+ 序列化给前端运行时联动，支持 **fetchApi 型**和 **state 型**
- 前端渲染引擎：嵌套路径拍平为 dot 路径，注册表支持 **35+** 组件动态渲染，发布订阅模式执行联动
- Monorepo 治理：Lerna + Nx + Yarn workspace 管理 **15+** 业务模块，独立版本 + 构建缓存

**结果**：新增活动类型前端 **零代码**，上线周期从 **2 周缩短到 3 天**，累计支撑 **20+ 种活动类型**落地，联动配置复用率 **80%+**。

### 营销权益组件库 — 400+ 权益 React 组件库

*架构设计 + 核心开发 · React · TypeScript · Rollup · Changesets*

**背景**：营销平台管理 400+ 种营销权益（优惠券、折扣、积分、立减等）的全生命周期，权益类型持续增长，原有方案每新增一种权益都需要硬编码，维护成本高。

**行动**：
- Register 注册模式：内置 **24** 组件 + 运行时注册自定义权益组件，新增权益无需修改 SDK
- Schema 同构渲染：组件库与活动表单共享 Schema 渲染架构，两个消费者复用同一套架构
- Rollup 双模打包（ESM + CJS + 类型声明），peerDependencies 避免依赖爆炸，Changesets 语义化发版

**结果**：覆盖 **8 个业务项目**，累计接入 **400+ 种权益**，新权益接入从 **2 天缩短到 2 小时**，组件库零 breaking change 运行 **18 个月+**。
