# AI 工程师能力图谱与准备路线（2026 市场版）

## 一、市场全景：三大方向 + 你的位置

### 1. AI 工程三大方向（2026）

| 维度 | 算法工程师 | AI Infra 工程师 | **AI 应用工程师（你）** |
|------|-----------|----------------|----------------------|
| **核心工作** | 训练 / 改进模型本身 | 支撑训练与推理的底座 | 把模型能力封装成可交付产品 |
| **技术栈** | 数学 + DL + PyTorch，顶会论文 | C++/CUDA + 分布式 + 推理优化（vLLM/TensorRT） | **Python + LangChain/Dify + RAG + Agent + Prompt** |
| **门槛** | ⭐⭐⭐⭐⭐ 最高（科班硕博） | ⭐⭐⭐⭐ 较高（C++/系统底子） | ⭐⭐⭐ 中等（后端/全栈可转） |
| **市场需求** | 大厂核心岗，K 型分化 | 稀缺稳定，年龄友好 | **全行业通用，需求量最大** |
| **薪资带** | 初级内卷 100:1，资深 80-150W | 25-120W | 12-50W，大厂 Agent 核心 **70-115W** |

> 公式：**算法**出模型 → **Infra**搭台子 → **应用**做产品。你的主战场在第三层，但要懂第二层的原理（推理优化是大厂送命题）。

### 2. 你在哪：AI 应用工程师（Agent 方向）

一句话：**用 Agent + 全栈工程能力，把大模型变成可上线的产品**。不是纯前端，不是纯算法，是能端到端交付 AI 体验的工程型选手。这正是 [蚂蚁财富 AI Lab](定向/蚂蚁财富AI-Lab.md) 这类岗位的本质。

### 3. 市场分层（数百份 JD 抽样）

| 分层 | 代表岗位 | 对口度 |
|------|---------|-------|
| 算法研究类 | 科大讯飞/商汤/字节研究院（NLP/多模态/Agent 研究员） | ❌ 硕博科班，不碰 |
| **AI 工程/开发类** | 腾讯 AI 工程架构师、美团大模型应用后端、字节 AI 后端 | ✅ **需求量最大，主投** |
| **Agent 专项** | 华为 AI 智能体专家、蚂蚁 PaaS 智能体平台、字节多 Agent 框架 | ✅✅ **最热风口，你的王牌区** |
| AI 产品/解决方案 | 字节 AI 智能体商业化、腾讯 AI PM | 🟡 备选（偏业务） |
| AI 基础设施/运维 | 旷视 K8s SRE、商汤云原生架构 | ❌ 偏运维/系统 |

### 4. 关键趋势（直接影响备考重心）

1. **应用层 > 基础层**——JD 从"懂 AI"转向"能落地交付"，纯 demo 级已无竞争力。
2. **Agent 是最大风口**——出现频率最高的关键词，多家专设 Agent 岗。
3. **算法 + 工程两手硬**——纯调参侠 / 纯理论派在淘汰，市场要复合型人才。
4. **2026 面试重心：大模型基础 → AI 应用落地**——Agent / MCP / RAG 全链路 / eval 成为新高频。

## 二、能力图谱：市场要求 vs 你的现状

### 1. 市场技能栈密度排序（JD 关键词频次）

```
Python(必备) → LLM 原理 + Prompt + RAG + Fine-tuning + 模型评估
```

## 三、三大王牌 × 四块短板

### ✅ 王牌（放大，每场必讲）

1. **Agent 工作流**：不是单点工具，是 PRD → 可执行开发计划的端到端蒸馏，靠**证据链 + 质量门控**保证可靠性。
2. **Prompt + 上下文工程**：XML 约束 + 中间文件传状态 + 渐进式披露 + 上下文预算（0-40% Smart Zone）。
3. **Tool Use / MCP**：分层路由（Router 分类）+ 动态激活 + 描述质量优化——几十个工具识别率不塌的解法。
4. **RAG 全链路**：倒排 + BM25 + 语义混合 + RRF 融合 + cross-encoder 重排，前端做向量化预处理 + 反馈闭环。

### ⚠️ 短板（补齐到门槛，2-3 周可达成）

| 短板 | 目标深度 | 怎么补 |
|------|---------|-------|
| **Python 工程化** | 能写生产级异步、数据处理、虚拟环境 | 有 JS 基础学很快；重点补 asyncio / NumPy·Pandas / FastAPI |
| **模型底层原理** | 能讲清，不必会训 | Transformer 结构 / Attention / **GQA** / KV Cache / 量化 / LoRA 概念 |
| **流式输出深度** | 5 层细节脱口而出 | 连接（fetch+ReadableStream）+ 解析（chunk buffer）+ 渲染（rAF 节流 + 增量 Markdown + 代码块保护）+ 可靠性（断线重连 + Last-Event-ID）+ 性能（虚拟列表 + Worker） |
| **eval 体系** | 把你的质量门控讲成评测方法论 | RAG 三维评测（召回/准确/相关）+ Agent 五维评分 + 确定性脚本 + 人工对比 |

### 🎯 兜底（不崩即可，不投入过多）

- **C 端 / 移动端**：主动承认 B 端为主，强调 Schema 引擎场景可迁移 + 准备 2-3 个 C 端故事（性能 / 设计稿还原 / 移动端兼容）。
- **多模态 / 推理优化**：了解概念（GQA / KV Cache 解决什么），被问到能讲原理即可。

## 四、准备路线（5 阶段，对接已有资产）

> 每阶段标注**对接资产**，不重复造轮子。日常看本文 + 题库，临考看急救包，面试中看 [stealth](../interview-tools/stealth.html)。

### 阶段 0 · 定位校准（1 天）

- [ ] 简历锚定「**AI 工程 + 前端全栈差异化**」，对齐 [简历 v3](简历/v3.md) 的项目数据口径（60% / 50% / 200+）
- [ ] 投递策略：主投 AI 工程/开发类 + Agent 专项（见第五节投递矩阵），避开算法研究岗
- [ ] 故事化定位句："4 年前端 + Schema 全栈，用 Agent 工作流把大模型变成可上线产品"

### 阶段 1 · 放大王牌（1-2 周）

- [ ] [deepdive-prd-tools](../interview-tools/deepdive-prd-tools.md) 逐题口述 2 遍，证据链 / 质量门控 / 反馈回流能 90 秒讲清
- [ ] **MCP 底层原理补强**（2026 新高频）：协议架构 / 状态管理 / 通信机制 / 与 Function Calling 对比——你已有实践，补"为什么这么设计"
- [ ] Agent 表达框架：多 Agent vs 单 Agent 选型 / 稳定性 / 衡量效果三层——见 [蚂蚁文档 A 组](定向/蚂蚁财富AI-Lab.md)

### 阶段 2 · 补齐高频短板（2-3 周，最关键）

- [ ] **流式输出 5 层**：题库已有，加深到生产级（断线重连 / 去重 / 万级 token 渲染 / 中断 AbortController）
- [ ] **Transformer + 推理机制**：Attention / GQA / KV Cache / 量化 / LoRA——原理级，能讲"解决什么、怎么 trade-off"
- [ ] **Python 工程化**：asyncio + FastAPI 写一个 LLM API 服务 demo（你已有 Node BFF 经验，迁移成本低）
- [ ] **eval 体系**：把 prd-tools 的五维评分 + 确定性检查 + 人工对比，提炼成"AI 应用评测方法论"

### 阶段 3 · 系统设计 + 算法手撕（1-2 周）

- [ ] AI 系统设计 4 题：智能客服前端 / Agent 理财顾问 / Schema 动态页 / 团队级 AI 编程工作流（见 [蚂蚁文档 F 组](定向/蚂蚁财富AI-Lab.md)）
- [ ] LeetCode 中等高频：树 / 动规 / 双指针 / 字符串——每天 1-2 题保持手感

### 阶段 4 · 定向冲刺（每岗 2-3 天）

- [ ] 复用 [蚂蚁财富 AI Lab](定向/蚂蚁财富AI-Lab.md) 模板：岗位本质 → 匹配度 → 高频问题预测 → 风险预案 → 反问 → 24h checklist
- [ ] stealth.html 加公司专属卡（emer + tag，参考 loop 配置）
