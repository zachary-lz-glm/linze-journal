## 四、手撕代码（大麦真题 + 阿里高频）

### 第一梯队（大麦真题，必会）

#### 题 1：千分位逗号

```javascript
// 方法 1：正则（面试官期望）
function format(num) {
  return String(num).replace(/\B(?=(\d{3})+(?!\d))/g, ',');
}
// \B 非单词边界；(?=(\d{3})+(?!\d)) 正向预查：后面是 3 的倍数连续数字且后面不再是数字

// 方法 2：循环（容易讲清楚）
function format(num) {
  let str = String(num);
  let result = '';
  while (str.length > 3) {
    result = ',' + str.slice(-3) + result;
    str = str.slice(0, -3);
  }
  return str + result;
}

console.log(format(1234567));    // "1,234,567"
console.log(format(123));        // "123"
console.log(format(1234));       // "1,234"
```

```javascript
function extract(str) {
  const matches = str.match(/\(([^)]*)\)/g);  // 匹配 (内容)
  return matches ? matches.map(s => s.slice(1, -1)) : [];
}
// \( 左括号；([^)]*) 捕获组：任意非右括号字符；\) 右括号

// 方法 2：栈遍历（候选人思路）
function extract(str) {
  const stack = [];
  const result = [];
  for (const ch of str) {
    if (ch === '(') stack.push(ch);
    else if (ch === ')') {
      if (stack.length) {
        const content = stack.slice(1).join('');
        result.push(content);
        stack.length = 0;
      }
    } else if (stack.length) stack.push(ch);
  }
  return result;
}

console.log(extract('a(b)c(d(e)f)g'));  // ['b', 'e']  注意：不匹配嵌套
```

### 第二梯队（阿里高频）

| 题目 | 核心思路 |
|------|---------|
| 防抖（debounce） | `clearTimeout + setTimeout`，只执行最后一次 |
| 节流（throttle） | flag 锁，固定频率执行 |
| 深拷贝（处理循环引用） | WeakMap 记录已拷贝对象，递归 |
| Promise.all | 计数器 + 结果数组，全部 fulfilled 才 resolve |
| LRU 缓存 | `Map` 保持插入顺序，get/put 时 delete+重新 set 移到末尾 |

**⚠️ 千问一面翻车点：防抖节流比喻说反**（Q38）
- **防抖** = 玩英雄联盟回城——每次按都重新读条，**只有最后那次能完成**
- **节流** = 技能冷却——用完后**固定 CD 内不能再放**，但 CD 到了就一定能放

### 练习优先级

1. **今天必练**：千分位 + 提取括号（大麦真题，会考）
2. **明天**：防抖节流（写熟 + 比喻背熟）+ 深拷贝
3. **面试前**：Promise.all + LRU

### 6.3 场景设计题（基于大麦业务）

#### 场景 1：设计抢票页面的前端架构

**核心思路（四层）**：
1. **排队层**：进入页面先排队（WebSocket 长连 + 队列位置实时推送），避免瞬时打爆后端
2. **库存层**：前端展示剩余票数（WebSocket 推送 + 节流更新，避免抖动）
3. **下单层**：点击购买 → 防抖（500ms）→ 二次确认 → 提交订单
4. **降级层**：网络异常 → loading 优化 → 重试引导 → 兜底静态页

**关键细节**：
- 防 URL 直接刷：进入页面先校验 token + 时间窗口
- 防超卖：前端只做展示，下单结果以服务端为准（避免乐观更新）
- 防机器人：行为埋点 + 风控 SDK

#### 场景 2：设计演出详情页（SEO + 性能）

**核心思路**：
1. **SSR/预渲染**：演出详情对 SEO 敏感，首屏走 SSR
2. **结构化数据**：JSON-LD 标注演出信息（Schema.org），提升搜索富文本展示
3. **图片优化**：WebP/AVIF + CDN 自适应尺寸 + 懒加载
4. **视频懒加载**：预告片 lazy embed，点击才加载播放器
5. **长性能预算**：LCP < 2.5s，FID < 100ms

### 6.4 AI Coding 方向额外准备（大麦属于阿里体系，AI 是加分项）

**必背 3 个概念**：

1. **Harness Engineering**（小红书二面考过，候选人答得浅）
   - Agent = Model + Harness
   - Harness 是为模型搭建约束、反馈、记忆、技能进化的全套控制架构
   - 三类引擎：合并引擎、通信引擎、反思引擎
   - KSG（Knowledge-Skill-Growth）：短期记忆→长期记忆→知识库→技能手册的抽象提取链路

2. **MCP vs Function Calling**（小红书二面 Q8 已答对）
   - Function Calling：模型内置、绑定具体厂商、单次调用
   - MCP：Anthropic 开放协议、跨模型复用、标准化工具发现机制

3. **阿里淘天工业级 AI 研发体系**（千问一面+阿里一面都提到）
   - PRD → Coding Agent → 知识图谱 → Harness 验证 → Agent Loop 自修复 → CI/CD
   - 我的 prd-tools 是这个体系的 PRD→Plan 阶段雏形
