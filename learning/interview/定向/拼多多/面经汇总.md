# 拼多多前端面经汇总

> 汇总自牛客网、LeetCode、掘金、CSDN 等平台 2018-2025 年面经
> 适用于高级前端开发工程师（广告投放 / 低代码 / 营销方向）

---

## 一、面试流程与风格

### 1.1 流程结构（3-4 轮）

| 轮次 | 面试官 | 时长 | 核心内容 | 特点 |
|------|--------|------|---------|------|
| 一面 | 技术 | 45-60min | 前端基础 + 手撕代码 + 简单算法 | 覆盖面广，JS/CSS/React/网络都会问 |
| 二面 | 技术/组长 | 45-60min | 项目深挖 + 场景设计 + 中等算法 | 连续追问"为什么"，不接受泛泛而谈 |
| 三面 | 总监/VP | 30-45min | 架构视野 + 技术深度 + 偶尔手写 | 可能问设计题、抗压测试 |
| HR 面 | HRBP | 20-30min | 动机、薪资、团队协作、文化适应 | 会确认能否接受高强度节奏 |

### 1.2 面试风格特点

- **节奏快**：问题密集，一个接一个，不给太多思考时间
- **追问深**：每个回答至少追问 2-3 层，直到你说"不确定"
- **手写必考**：每轮至少一道手撕代码，大概率是 JS 手写而非纯算法
- **项目主导**：二面三面围绕项目展开，设计决策要有理有据
- **不排斥重复**：不同轮次可能问相似问题，考察一致性

### 1.3 时间线参考

- 社招（1-3 年）：一轮到 HR 面通常 1-2 周
- 社招（3-5 年）：可能加一轮交叉面
- 面试时间安排：工作日晚间或周末

### 2.5 手撕代码题（一面重点）

**第一梯队：几乎必考（至少出现一道）**

1. **深拷贝（deepClone）**
   - 基础版：递归处理对象和数组
   - 进阶版：处理循环引用（WeakMap）、Symbol、Date、RegExp、Map/Set、函数
   - 边界：null、基本类型、数组、嵌套对象

2. **防抖（debounce）和节流（throttle）**
   - debounce：延迟执行，重复触发重置计时器
   - throttle：固定间隔执行，避免高频触发
   - 进阶：leading/trailing 选项、取消方法

3. **Promise 实现**
   - Promise.all：全部成功才 resolve，一个失败就 reject
   - Promise.race：返回最先 settle 的结果
   - Promise.allSettled：所有结果都返回，不因失败中断
   - Promise 并发控制：限制同时执行的 Promise 数量

4. **发布订阅（EventEmitter）**
   - on / off / emit / once
   - 进阶：命名空间、once 自动移除、参数传递

**第二梯队：高频出现**

5. **URL 参数解析**
   - `?a=1&b=2` → `{ a: '1', b: '2' }`
   - 处理重复 key、编码解码、hash

6. **数组扁平化（flatten）**
   - 递归 + reduce
   - 指定深度 flat(depth)

7. **对象数组转树结构**
   - 给定 `[{id, pid, name}]` 构建树
   - Map 优化 O(n) 方案

8. **reduce 手写实现**
   - accumulator + currentValue + index + array

9. **call / apply / bind 实现**
   - context 绑定 + 参数传递

10. **new 操作符实现**
    - 创建对象 → 绑定原型 → 执行构造函数 → 返回

**第三梯队：偶尔出现**

11. **EventEmitter 升级版**：支持命名空间、通配符
12. **LRU 缓存**：get/set O(1)，Map + 双向链表
13. **异步串行执行器**：按顺序执行 Promise
14. **函数柯里化**：`add(1)(2)(3)` → 6
15. **async pool（异步并发控制）**：同时最多 N 个请求

## 三、二面：项目深挖 + 场景设计 + 中等算法

### 3.1 项目深挖模式

拼多多二面最核心的部分是项目深挖，典型追问链如下：

```text
第一层：你做过什么项目？核心职责是什么？
第二层：为什么选择这个技术方案？有没有考虑过其他方案？
第三层：这个方案最大的难点是什么？怎么解决的？
第四层：方案有什么不足？如果重做会怎么改进？
第五层：（数据指标）效果怎么样？有没有量化数据？
```

**典型追问方向**：

| 追问类型 | 面试官话术 | 应对策略 |
|---------|----------|---------|
| 技术选型 | "为什么用 X 不用 Y？" | 给出 2-3 个方案的 trade-off 对比 |
| 架构设计 | "整体架构是怎样的？画一下" | 分层画图，说清数据流 |
| 难点挑战 | "这个项目最难的地方在哪？" | 准备 2-3 个具体难点 + 解决过程 |
| 团队协作 | "你和后端/产品怎么配合的？" | 举具体例子，说清沟通方式 |
| 量化成果 | "优化前后数据对比？" | 诚实回答，没有数据就描述定性改进 |
| 改进空间 | "如果重做会怎么改？" | 1-2 个具体改进点，展示反思能力 |

### 3.2 场景设计题

**高频设计题**：

1. **设计一个 Input 组件（TypeScript）**
   ```typescript
   // 面试官期望你写出：
   interface InputProps {
     value: string;
     onChange: (value: string) => void;
     placeholder?: string;
     disabled?: boolean;
     maxLength?: number;
     prefix?: ReactNode;
     suffix?: ReactNode;
     status?: 'error' | 'warning' | 'default';
   }
   // 需要考虑：受控/非受控、事件处理、样式隔离、可访问性
   ```

2. **手写 Redux createStore（TypeScript）**
   ```typescript
   // 核心接口
   function createStore<S>(reducer: Reducer<S>, initialState?: S): Store<S>
   // 需要实现：getState / dispatch / subscribe / unsubscribe
   ```

3. **设计一个低代码建站平台**
   - 数据层：JSON Schema 描述页面结构
   - 引擎层：渲染引擎 + 组件注册表
   - 编辑层：拖拽 + 属性面板 + 预览
   - 扩展层：插件 / 脚本 / 自定义组件

4. **图片懒加载实现**
   - IntersectionObserver 方案
   - 滚动监听 + getBoundingClientRect 方案
   - 优化：节流、占位图、loading 状态

5. **倒计时组件**
   - 精确计时（Date 差值而非 setInterval 累加）
   - 防篡改系统时间
   - 性能：隐藏时暂停、requestAnimationFrame

6. **图片重试组件**
   - React/Vue 实现
   - 最大重试次数 + 退避策略
   - loading / error / success 状态

7. **Promise.race + 超时重试**
   ```javascript
   // 面试官原题：实现带超时和重试的请求
   function fetchWithRetry(url, options, retries = 3, timeout = 5000)
   // 考点：Promise.race 超时控制 + 递归重试 + 错误处理
   ```

| 类型 | 问题 |
|------|------|

- **到岗时间**：能多快入职

## 七、备战优先级排序
