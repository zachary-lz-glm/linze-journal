## 核心备战方向（按优先级）

### P0：必考题（1-3 天准备）

**1. 低代码建站平台设计题**
- 面试官大概率问："如何设计一个 H5 低代码建站平台？"
- 我的回答框架：
  - 数据层：Schema 描述页面结构（JSON 配置化）
  - 引擎层：渲染引擎解析 Schema → 组件注册表匹配 → 动态渲染
  - 联动层：字段间联动用发布订阅 + 双重求值（SSR + CSR）
  - 扩展层：Register 模式支持运行时注册新组件
  - 工程化：Monorepo 管理多模块，模板引擎生成 Schema

**2. 组件抽象与设计模式**
- Register 注册模式 vs 插件模式 vs 策略模式
- Schema 同构渲染（组件库与表单共享渲染引擎）
- Rollup 双模打包 + peerDependencies 依赖隔离

**3. React 原理深挖**
- Fiber 架构 + 时间切片 + 优先级调度
- Hooks 闭包陷阱 + 为什么不能条件调用
- useEffect / useLayoutEffect / useInsertionEffect 执行时机

**4. 算法高频题（拼多多偏好）**
- 二叉树：层序遍历、Z 字形遍历、最近公共祖先
- 动态规划：最长递增子序列、零钱兑换
- 字符串：最长回文子串、无重复字符最长子串
- 手写：深拷贝、防抖节流、Promise.all、发布订阅

### P1：加分项（面试前 1 天突击）

**5. 性能优化实战**
- 大型 SPA 白屏优化（SSR / 预渲染 / 懒加载）
- Bundle size 优化（Tree Shaking / Code Splitting / 动态 import）
- 运行时性能（虚拟列表、Web Worker、requestAnimationFrame）

**6. 工程化体系**
- Monorepo 治理（Lerna + Nx + Yarn workspace 的坑和收益）
- CI/CD + 灰度发布 + 监控告警
- 组件库版本管理和 breaking change 防护

**7. 广告投放方向加分**
- 了解广告投放平台的基本概念：DSP、SSP、RTB、CTR
- H5 落地页性能指标：首屏时间、FCP、LCP
- A/B 测试与数据驱动优化
