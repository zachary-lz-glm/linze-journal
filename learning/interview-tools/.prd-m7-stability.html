◆ 实现
AI 生成的代码不稳定、输出飘怎么治
不指望 LLM 自己稳——能走模板就走模板(零 token、跑一万次一样),走不了才交 LLM,再叠 4 类校验 + 3 次差异重试 + 震荡检测兜底

根因是 LLM 本身非确定,不靠调 prompt 解决。同输入不同次跑出不同质量是模型特性,治法是确定性优先:有 Handlebars 模板覆盖的 campaign 类型走模板快速路径(源码注释原话"zero LLM Token consumption",跑一万次结果一样),只有无模板匹配时才走 LLM 兜底路径。把大部分确定性场景用模板兜走,LLM 只处理长尾。

模板覆盖不到的,靠校验加重试收敛。bff-gen step-05 跑 4 类确定性校验——syntax(语法)/ fields(字段)/ rules(规则)/ integration(集成),每类是 src/validators/ 下的独立脚本。校验不过自动重试,最多 3 次,每次只修报错字段或行而非全量重写,修完跑全量验证;3 次仍不过暂停交人工,不无限循环。

震荡检测防反复横跳。修正历史记到 validation.json 的 history 数组,识别 A→B→A 的反复修正模式(改过去又改回来)后切换策略——不再逐字段修,直接整段重写,避免在两个状态间无限震荡。这是把"改了又改"的隐蔽退化转成显性触发条件。

为什么不报"识别率"类指标。prd-tools 是纯 Skill 形态、不注册一堆外部 function-calling 工具给模型选,所以"工具多了模型选错"这类问题在本项目根本不存在;输出稳定完全靠上面的确定性优先 + 校验重试 + 震荡检测,不混入无关的外部研究数据。
